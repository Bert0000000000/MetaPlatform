P0 修复中 BUG-0042

客户表单提交后数据丢失

在客户管理页面中，当用户填写客户信息表单并点击提交后，部分字段数据未正确保存到数据库。具体表现为：联系电话和备注字段在提交后显示为空值，但其他字段（客户名称、类型）正常保存。

模块 客户管理
类型 数据丢失
AI置信度 92%
复现步骤
  1. 进入客户管理页面，点击"新增客户"按钮
  2. 填写客户名称、客户类型、联系电话（13800138000）、备注（测试备注信息）
  3. 点击"保存"按钮提交表单
  4. 返回客户列表，找到刚创建的客户记录
  5. 点击编辑，观察联系电话和备注字段是否为空
AI 修复建议
根因分析： 表单提交接口 /api/customer/create 的请求体中缺少 phone 和 remark 字段映射。
修复方案： 在 CustomerFormMapper.java 中补充字段映射关系，并在 DTO 校验中添加非空约束。
修复前后对比
修复前
@PostMapping("/create")
public Customer create(CustomerDTO dto) {
  return service.save(dto);
}
修复后
@PostMapping("/create")
public Customer create(@Valid @RequestBody CustomerDTO dto) {
  Customer entity = mapper.toEntity(dto);
  entity.setPhone(dto.getPhone());
  entity.setRemark(dto.getRemark());
  return repository.save(entity);
}
