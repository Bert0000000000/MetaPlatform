# MetaPlatform 菜单产品设计 v3（决策版）

> **文档版本**：v3（决策版）
> **核心结构**：**一级菜单 12 项 + Tab 深度优化 + 三层 AI 入口**
> **设计理念**：**场景驱动 × 角色化分发 × 扁平聚合**
> **更新日期**：2026-07-03
> **核心调整**：基于用户决策，**自由页面**与**门户**合并到 **Dashbaord** 作为 Tab；**服务编排**合并到 **流程中心** 作为 Tab

---

## 〇、v2 → v3 关键调整

### 0.1 用户决策

| 决策项 | 详情 |
|--------|------|
| ✅ 1. **自由页面** 合并到 **Dashbaord** | 不独立成一级菜单，作为 Dashbaord 下 Tab |
| ✅ 2. **服务编排** 合并到 **流程中心** | 业务流程 / 审批流程 / 服务编排 统一在流程中心 |
| ✅ 3. **门户** 合并到 **Dashbaord** | PC+移动端入口作为 Dashbaord 下 Tab |

### 0.2 v3 vs v2 菜单结构对比

| 模块 | v2 | v3 |
|------|----|----|
| Dashbaord | 4 个 Tab | **6 个 Tab**（+ 门户 + 自由页面） |
| 流程中心 | 7 个 Tab | **9 个 Tab**（+ 业务流程 + 审批流程 + 服务编排） |
| 一级菜单总数 | 12 项 | **12 项（不变）** |

### 0.3 调整理由

1. **保持 12 项一级菜单的认知舒适**
   - 一级菜单 ≤ 12 项是认知舒适区上限
   - 不到 1 年迭代不应轻易增加一级菜单
2. **聚合相似能力**
   - 自由页面是"我的工作空间"的一种能力，归入 Dashbaord 自然
   - 门户是登录入口的另一种形态，归入 Dashbaord 自然
   - 服务编排本质是"流程化的服务调用"，归入流程中心自然
3. **避免一级菜单碎片化**
   - 用户不需要为"门户"和"自由页面"单独记忆
   - 3 类流程（业务/审批/服务编排）统一管理更高效

**为什么合并自由页面到 Dashbaord？**
- 自由页面本质是"用户自定义的可视化页面"
- 与"我的应用""我的数字员工"同属消费侧
- Dashbaord 是千人千面的"我的空间"，自由页面正是"自定义空间"能力的体现

**为什么合并门户到 Dashbaord？**
- 门户是用户登录后的**统一入口**
- 与工作台互补（工作台是数据看板，门户是导航入口）
- 在 Dashbaord 下作为"导航入口 Tab"自然

## 五、v3 设计原则

| 原则 | 说明 |
|------|------|
| **1. 一级菜单 ≤ 12 项** | 12 项是认知舒适区上限 |

| **3. 相似能力合并** | 自由页面+门户 合并到 Dashbaord；服务编排 合并到流程中心 |
| **4. Dashbaord 是千人千面** | 工作台 + 我的应用 + 我的数字员工 + 门户 + 自由页面 |
| **5. 流程中心是 3 类流程聚合** | 业务流程 + 审批流程 + 服务编排流程 |
| **6. AI 独立顶级入口** | SuperAI 一级菜单，3 层 AI 入口 |
| **7. 管理功能进后台** | 后台管理独立成一级菜单 |
| **8. 数字员工差异化** | 一级菜单，强调 AI 原生应用 |
