# 文档结构规范

## 目录组织

```
docs/
├── 000-GUIDE/       # 规范与指南：命名规范、文档结构、编码规范
├── 001-ARCH/        # 架构设计：应用架构、技术架构、数据架构、架构图
│   └── assets/      # 架构图资源
├── 002-TS/          # 技术选型：技术栈选型、版本信息、对比分析
├── 003-SPEC/        # 规范与接口：API 规范、Ontology 元模型、协议规范
├── 004-PLAN/        # 计划与路线图：实施计划、里程碑、版本规划
├── 005-RD/          # 调研报告：技术调研、竞品分析、PoC 报告
└── 006-TMP/         # 临时脚本：生成脚本、调试工具、中间产物
```

## README 要求

每个目录（包括项目根目录、APP/TECH 模块目录、docs 子目录）必须包含 `README.md`，内容包含：

1. **模块/目录名称**
2. **作用说明**：该模块/目录的职责
3. **上下游关系**：上游依赖、下游消费
4. **目录结构**：内部文件组织方式
5. **相关文档链接**

## 文档版本管理

- 每份文档在文件名中标注版本和日期：`v1.0-20260716`
- 文档内容头部应包含版本历史：

```markdown
## 版本历史

| 版本 | 日期 | 变更说明 | 作者 |
|---|---|---|---|
| v1.0 | 2026-07-16 | 初始版本 | - |
```
