# MetaPlatform v1.2 模块架构优化规划（基于百特搭架构图 v3）

> **文档版本**：v1.2（基于 v1.1 升级）
> **核心依据**：百特搭 AI 原生应用平台产品架构图 + MetaPlatform 现有实现
> **核心目标**：从"按技术层划分"升级为"按用户角色 × 应用场景"划分，让 MetaPlatform 真正成为**面向企业 AI 智能化的 AI 原生应用平台**
> **更新日期**：2026-07-03

---

## 〇、与 v1.1 的关键差异

| 维度 | v1.1 | v1.2（基于架构图升级） |
|------|------|----------------------|
| **顶层入口** | 一个工作台 | **4 类触点端**：领导工作台 / 一站式业务大厅 / 移动端 IM / AI 原生应用 |
| **核心引擎** | 7 个并列模块 | **3 大引擎 + 2 个支撑平台**：融合构建引擎、决策引擎、多智能体协作引擎 + 质量验证平台 + 本体引擎 |
| **本体建模** | ObjectType + 字段 + 关系 | **本体 8 要素**：对象 / 属性 / 关系 / 动作 / 函数 / 流程规则 / 安全 / 治理与发布 |
| **AI 原生应用** | 在"应用中心"下 | **独立顶层分类**：「AI 原生应用系统（职能/业务）」与"客户/企业"应用并列 |
| **多智能体** | 在 SuperAI 下 | **独立引擎层**：「多智能体协作 / 数字员工引擎」+ 智能体决策会议 + 身份与记忆 |
| **数字员工** | 无 | **独立概念**：Agent 按"数字员工"方式创建 / 管理 / 组织，含人格、长期记忆、独立工作空间 |
| **VibeCoding** | 无 | **与低代码并列**：「VibeCoding 平台」与 LowCode 工具集双轨 |
| **构建方式** | 单一拖拽 | **三轨**：纯代码 / 可视化配置 / VibeCoding AI 生成 |
| **侧边栏** | 单一侧边栏 | **按角色分侧边栏**：领导 / 业务开发人员 / 运维人员 / AI 使能 |
| **应用市场** | 应用市场 | **6 类模板**：本体行业模板 / Skill / Agent / 工作流 / 知识包 / API 动作 |
| **底层基础设施** | 隐式 | **显式 3 大底座**：非 AI 原生系统 / RAG 非结构化知识库 / 部署及环境（单机·集群·私有化·多租户·沙盒） |

## 一、百特搭架构图核心设计理念

### 1.1 一句话定位

> **致力于企业 AI 智能化转型 —— AI 原生应用平台**

MetaPlatform 对应定位：**本体论驱动的 AI 原生应用平台，让每一家企业拥有自己的数字员工团队。**

### 1.2 4 类用户角色 × 4 类触点端

```

├─────────────┬───────────────────────────────────────────────────────┤

├─────────────┼───────────────────────────────────────────────────────┤

│ 🔧 开发人员  │ VibeCoding 平台 · LowCode 低代码工具集                │
```

### 1.3 5 层堆叠架构（核心骨架）

```
┌─────────────────────────────────────────────────────────────────┐
│  L0 触点端  4 类入口 + IM 集成                                    │
├─────────────────────────────────────────────────────────────────┤
```

### 模块 1：🏠 数字工作台（替代 v1.1 "工作台"）

**定位**：4 类用户登录后的**第一屏**，千人千面
**关键变化**：从"通用 Dashboard"升级为"角色化工作台"

```
数字工作台
├── 领导工作台（👑 高管/部门负责人）
│   ├── AI 看板（关键指标卡片 + 趋势图）
│   ├── AI 问数（自然语言查数据）
│   ├── 团队数字员工概览
│   ├── 待审批 / 待决策
│   └── 战略指标进度
├── 业务工作台（💼 业务人员）
│   ├── 一站式业务大厅入口（PC/移动）
│   ├── 我的待办（流程/工单/审批）
│   ├── 我的应用（最近使用/收藏）
│   ├── 信息门户（公告/通知/知识）
│   └── 数字员工协同（我的 AI 助手）
├── 开发工作台（🔧 业务开发人员）
│   ├── 我的应用（开发中的应用）
│   ├── 我的智能体（开发中的 Agent）
│   ├── 我的本体（建模进度）
│   ├── FDE 工作台快捷入口
│   └── VibeCoding 快捷入口
└── 运维工作台（⚙️ 运维人员）
    ├── 系统健康总览（服务/数据库/中间件）
    ├── 告警中心（待处理告警）
    ├── 用户活跃度
    ├── 资源使用率
    └── 快速运维入口（重启/扩容/备份）
```

**对应后端**：
- 领导工作台 → `/api/v1/dashboard/executive?tenantId=...`
- 业务工作台 → `/api/v1/dashboard/business?userId=...`
- 开发工作台 → `/api/v1/dashboard/developer?userId=...`
- 运维工作台 → `/api/v1/dashboard/ops`

**定位**：v1.1 是"独立 AI 入口"，v1.2 升级为**嵌入式 AI 助手**（全局可唤起）+ **集中 AI 工作台** 双形态

```
SuperAI
├── 全局 AI 助手（⌘K 唤起，悬浮式）
│   ├── 自然语言对话
│   ├── 上下文记忆（跨模块）
│   ├── 多模态输入（图片/语音/文档）
│   ├── 智能跳转（识别意图后跳到对应模块）
│   └── 快捷动作（建应用/建模/查数据/起流程）
├── AI 工作台（集中式）
│   ├── 对话历史
│   ├── 我的 AI 任务
│   ├── 智能体调用记录
│   └── Token 用量
├── AI 建模助手（架构中心嵌入）
│   ├── NL → ObjectType 生成
│   ├── 字段/关系建议
│   ├── 本体检查（Linter）
│   └── 智能补全
├── AI 流程助手（流程中心嵌入）
│   ├── NL → 流程 DSL
│   ├── 流程解释（NL 解释）
│   ├── 智能派单
│   └── SLA 预警
├── AI 数据助手（数据中心嵌入）
│   ├── NL → SQL
│   ├── 数据质量分析
│   ├── 异常检测
│   └── 指标推荐
└── AI 设置
    ├── 模型配置（OpenAI/Claude/DeepSeek/Qwen）
    ├── Token 用量 + 预算熔断
    └── API Key 管理
```

**关键升级**：每个模块内嵌 AI 助手按钮，而非跳转独立页面。

### 模块 3：🧠 决策引擎（v1.1 没有，独立模块）

**定位**：**百特搭第二大引擎** —— 让"业务指标 × AI 问数 × 业务推动"成为闭环

```
决策引擎
├── 个人工作台（用户级）
│   ├── 我的指标
│   ├── 我的报表
│   └── 我的待办
├── 智能问数
│   ├── 自然语言提问
│   ├── 智能 SQL 生成（基于本体）
│   ├── 结果可视化（自动选图表）
│   ├── 追问与下钻
│   └── 收藏与分享
├── 指标定义
│   ├── 原子指标（基于业务对象）
│   ├── 派生指标（公式组合）
│   ├── 维度定义
│   ├── 血缘追踪
│   └── 指标版本管理
├── 业务推动（行动闭环）
│   ├── 异常指标 → 自动建工单
│   ├── 指标达成 → 自动通知
│   ├── 决策建议（AI 推荐）
│   └── 复盘报告（自动生成）
└── 决策权限
    ├── 指标查看权限
    ├── 指标定义权限
    └── 行动触发权限
```

**对应后端**：
- 智能问数 → `NlSqlService` + `MetricQueryService`
- 指标定义 → `MetricDefinitionService` + `DorisWarehouseService`
- 业务推动 → `ActionTriggerService` + `WorkflowIntegration`

**这是 MetaPlatform 当前最缺的模块！v1.1 没有，需要 v1.2 新建。**

**定位**：百特搭最具差异化的部分 —— **Agent 不是工具，是数字员工**

```
多智能体协作 / 数字员工引擎
├── 智能体决策会议
│   ├── 会议创建（基于议题）
│   ├── 角色协作（主席/委员/记录员）
│   ├── 发言轮次控制
│   ├── 会议纪要自动生成
│   ├── 提案与行动项
│   └── 决策确认与归档
├── 多智能体协作
│   ├── Agent 间通信（消息总线）
│   ├── 任务分工（基于能力匹配）
│   ├── 委派与协同执行
│   ├── 协作工作流
│   └── 冲突解决
├── 身份与记忆
│   ├── 人格配置（角色/性格/语气）
│   ├── 长期记忆（用户偏好/历史交互）
│   ├── 知识记忆（行业知识/企业知识）
│   └── 工作记忆（当前任务上下文）
├── 工作空间
│   ├── 文件管理（Agent 私有空间）
│   ├── 代码管理
│   ├── 工具接入（MCP / 外部工具 / API）
│   └── 技能市场（Skill Marketplace）
├── 权限渠道
│   ├── RBAC 权限（与平台统一）
│   ├── 可见性控制
│   ├── 审批与审计日志
│   └── 渠道接入（Slack/飞书/钉钉/企微/Teams）
└── 模型及用量
    ├── 模型切换（OpenAI/Claude/DeepSeek/Qwen/自托管）
    ├── 用量监控（按 Agent / 按租户）
    ├── 预算熔断
    └── 部署运维（沙箱/Docker/K8s/企业化）
```

**关键差异化（vs v1.1）**：
- **数字员工**是一等公民 —— 有"人格/记忆/工作空间"，不是冷冰冰的 Agent
- **决策会议** —— 多 Agent 协同决策的新形态
- **工作空间私有化** —— 每个 Agent 有独立空间

**对应后端**：
- 智能体决策会议 → `AgentMeetingService` + `MeetingMinutesService`（**v1.2 新建**）
- 多智能体协作 → `AgentCollaborationService`（**v1.2 新建**）
- 身份与记忆 → `AgentIdentityService` + `LongTermMemoryService`（**升级现有 Agent**）
- 工作空间 → `AgentWorkspaceService`（**v1.2 新建**）
- 模型用量 → 升级现有 `TokenMeteringService`

- 业务规则 → 升级现有 `BusinessRuleService` + 新建 `DecisionTableService`

### 模块 6：🏗 架构中心（重大升级：基于"本体 8 要素"重构）

**定位**：**百特搭核心 —— 本体引擎驱动的统一建模**

**关键升级**：从 v1.1 的"对象/字段/关系/规则"4 维度，升级为**本体 8 要素**（对齐百特搭）：

```
架构中心（基于本体引擎 8 要素）
├── 1️⃣ 对象（Objects）
│   ├── 建模管理
│   ├── 增强描述
```

## 九、与百特搭的关键差异（MetaPlatform 差异化优势）

| 维度 | 百特搭 | MetaPlatform v1.2 |
|------|--------|------------------|
| 核心驱动 | 本体（8 要素） | **本体论（Ontology）+ 9 层架构** |
| AI 能力 | AI 原生应用 | **AI 基质（贯穿 9 层）** |
| 数据能力 | 隐式 | **显式 5 层数据栈**：连接器 / 仓库 / 湖 / 质量 / MDM / 血缘 |
| 知识能力 | RAG 知识库 | **RAG + GraphRAG + 知识包资源库** |
| 流程能力 | 流程中台 | **BPMN + AI 决策节点 + 流程挖掘 + 决策引擎联动** |
| 部署能力 | 单机/集群/私有化/沙盒 | **+ 信创适配（鲲鹏/麒麟/达梦/人大金仓）** |
| 数字员工 | Agent | **本体驱动的数字员工**（人格/记忆/工作空间全部本体化） |
| 智能体协作 | 多智能体 | **+ 决策会议 + 工作空间隔离** |
| VibeCoding | AI 生成代码 | **VibeCoding + LowCode + FDE 三轨** |
| 质量验证 | AI 测试 | **+ 本体模型一致性测试 + AI BUG 自动修复** |
| 数据底座 | 隐式 | **显式 PG/Neo4j/Milvus/MinIO/Doris/Hudi/ES 7 大存储** |
| API 治理 | API 动作库 | **+ GraphQL + REST + SSE 三协议统一** |

**MetaPlatform 核心定位升级**：

> **本体论驱动的 AI 原生应用平台 + 9 层技术栈 + 数字员工引擎 + 信创全栈适配**
>
> 致⼒于企业 AI 智能化转型（对齐百特搭）
