# MetaPlatform 菜单产品设计 v1（最终版）

> **文档版本**：v1（最终）
> **核心结构**：**一级菜单（侧边栏 4 项）+ 顶部 Tab 切换 + 全局 ⌘K AI**
> **设计理念**：**场景驱动 × 角色化分发 × 扁平化**
> **更新日期**：2026-07-03

---

## 〇、设计哲学

### 0.1 一句话总结

> **左：3 大场景空间 + 1 齿轮（侧边栏，固定 4 项）**
> **右：场景内 Tab 切换（顶部，动态加载）**
> **全局：⌘K AI 助手（任何页面唤起）**

### 0.2 为什么不只做一级菜单？

| 方案 | 优势 | 劣势 | 结论 |
|------|------|------|------|
| 12 个一级菜单平铺 | 直达 | 认知负担过重 | ❌ |
| 8 个一级菜单平铺 | 直达 | 业务聚合度不够（请假要跨 4 个菜单） | ❌ |
| **3 大场景 + 顶部 Tab** | **场景聚合 + 扁平导航** | **点 2 次到位** | ✅ |
| 3 大场景 + 页面内分组 | 极简 | 需要在页面内深度翻找 | ❌ |

**结论**：**一级菜单 + 顶部 Tab** 是平衡「场景聚合」和「扁平导航」的最优解。

## 一、整体架构

### 1.1 双层菜单视觉

```
┌──────────────────────────────────────────────────────────────────────────┐
│  MetaPlatform        [⌘K 全局搜索/AI]    [🔔 通知] [👤 角色切换]           │
├────────────┬─────────────────────────────────────────────────────────────┤
│            │                                                             │
│  🏠 我的工作 │  ┌─[ 工作台 ]─[ 我的应用 ]─[ 我的数字员工 ]─[ 消息 ]─┐   │
│  ──────    │  └────────────────────────────────────────────────────┘   │
│  🏗️ 构建应用 │  ┌─[ 应用 ]─[ 架构 ]─[ 流程 ]─[ 页面 ]─[ 智能体 ]─┐    │
│  ──────    │  └────────────────────────────────────────────────────┘   │
│  🧠 数据智能 │  ┌─[ 问数 ]─[ 指标 ]─[ 数据源 ]─[ 湖仓 ]─[ 知识 ]─┐    │
│  ──────    │  └────────────────────────────────────────────────────┘   │
│            │                                                             │
│  ⚙️ 设置    │            主内容区（当前 Tab 页面）                       │
│            │                                                             │
└────────────┴─────────────────────────────────────────────────────────────┘
```

### 1.2 一级菜单（侧边栏，固定 4 项）

| # | 一级菜单 | 图标 | 一句话 | 适合角色 |
|---|---------|------|--------|---------|
| 1 | **我的工作** | 🏠 | 我个人要用、要看的、要做的 | 全部 |
| 2 | **构建应用** | 🏗️ | 我要给团队/企业交付能力 | 业务+开发 |
| 3 | **数据智能** | 🧠 | 让数据和 AI 给业务带来价值 | 全部 |
| 4 | **设置** | ⚙️ | 平台配置、用户、权限、运维 | 运维（其他角色可见但只读） |

**为什么是 4 个？**
- **3 大场景** + **1 齿轮** = 认知舒适区（4 项 ≤ 7 项舒适阈值）
- **「设置」不进场景**：系统级配置和管理不进业务场景流，避免污染主流程

### 1.3 顶部 Tab 切换（场景内二级）

进入一级菜单后，顶部出现 Tab 切换，**Tab 数 ≤ 7 个**（认知舒适区）：

| 一级菜单 | 顶部 Tab |
|---------|---------|
| 🏠 我的工作 | 工作台 / 我的应用 / 我的数字员工 / 消息 / SuperAI（5 项） |
| 🏗️ 构建应用 | 应用 / 架构 / 流程 / 页面 / 智能体 / 模板市场（6 项） |
| 🧠 数据智能 | 问数 / 指标 / 决策 / 数据源 / 湖仓 / 知识 / 质量（7 项） |
| ⚙️ 设置 | 用户 / 权限 / 插件 / 服务网关 / 监控 / 备份 / 环境（7 项） |

### 🏠 场景 1：我的工作（My Workspace）

**核心理念**：千人千面，每个角色登录后看到的「工作台」完全不同。

```
我的工作
│
├── Tab 1: 工作台（按角色动态变化）
│   ├── 👑 领导版：AI 看板 + 战略指标 + 待审批 + 团队数字员工
│   ├── 💼 业务版：我的待办 + 最近应用 + 我的数字员工
│   ├── 🔧 开发版：开发中的应用 + 开发中的智能体 + FDE 入口
│   └── ⚙️ 运维版：系统健康 + 告警 + 资源使用率
│
├── Tab 2: 我的应用（消费侧）
│   ├── 已安装应用
│   ├── 最近使用
│   ├── 收藏应用
│   └── 团队应用推荐
│
├── Tab 3: 我的数字员工（消费侧）
│   ├── 我创建的数字员工
│   ├── 分配给我的数字员工
│   ├── 数字员工对话历史
│   └── 使用统计
│
├── Tab 4: 消息中心
│   ├── 待办事项（流程/审批/工单）
│   ├── @我的消息
│   ├── 系统通知
│   └── 数字员工主动推送
│
└── Tab 5: SuperAI（个人 AI 工作台）
    ├── 对话历史
    ├── 我的 AI 任务
    ├── Token 用量
    └── 模型偏好设置
```

**为什么这样设计**：
- 「工作台、我的应用、消息中心」**消费侧能力**放在一起，符合用户直觉
- 「数字员工」作为独立 Tab，强调 MetaPlatform 差异化
- 「SuperAI」是个人 AI 助手，与各业务场景的 AI 嵌入**两个层级**

### 🧠 场景 3：数据智能（Data & AI）

- **融合百特搭"决策引擎"+"数据中心"+"知识库"** 为统一场景

## 七、设计原则总结

### 7.1 7 条菜单设计原则

| 原则 | 说明 |
|------|------|
| **1. 场景驱动，非模块驱动** | 用户按"我要干什么"思考，不按"系统怎么分模块"思考 |
| **2. 一级菜单 ≤ 5 个** | 4 项（3 场景 + 1 齿轮），认知负担最小 |
| **3. 顶部 Tab ≤ 7 个** | 7 项以内是认知舒适区，超过考虑再分组 |
| **4. 角色化分发** | 不同角色看到不同菜单，不是同一菜单不同权限 |
| **5. AI 内嵌而非独立菜单** | AI 助手 ⌘K 全局可唤起，同时每个页面右下角浮窗 |
| **6. 管理功能进齿轮** | 平台配置/管理控制台等"系统级"功能不污染主菜单 |
| **7. 数字员工是一等公民** | 单独 Tab，强调差异化 |

### 7.2 与 v1.2 菜单对比

| 维度 | v1.2 | **v1（本设计）** |
|------|------|--------------|
| 一级菜单数 | 12 个（侧边栏） | **4 个（3 场景 + 齿轮）** |
| 二级展现形式 | 多级嵌套 | **顶部 Tab 切换** |
| 角色差异 | 同一菜单不同权限 | **不同角色不同菜单** |
| AI 入口 | 独立 SuperAI 模块 | **⌘K 全局 + 页面浮窗** |
| 管理功能污染 | 占一级菜单 | **进齿轮，不进主菜单** |
| 用户认知路径 | "请假" → 跨 5 菜单 | "请假" → 我的工作 → 我的应用 → 打开 |

## 八、验收标准

- [ ] 任何角色登录后，**主菜单不超过 4 项 + 齿轮**
- [ ] 每个场景顶部 Tab ≤ 7 个
- [ ] ⌘K 唤起 AI 命令面板，200ms 内显示
- [ ] 领导角色看不到"页面生成器"等开发菜单
- [ ] 业务用户能在一个场景内完成"请假申请"全流程（不用切到构建应用）
- [ ] 开发人员能在一个场景内完成"建模 → 流程 → 页面 → 智能体 → 发布"全流程
- [ ] 数字员工有独立入口（我的工作 Tab + 构建应用 Tab 都有）
- [ ] 平台配置不进一级菜单
