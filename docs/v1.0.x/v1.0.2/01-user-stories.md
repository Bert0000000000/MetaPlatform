## 一、Epic 拆分（按用户场景）

v1.0.2 拆为 **4 个 Epic（里程碑）**，每 Epic 完成有明确的 demo：

| Epic | 标题 | 主要用户故事 |
|------|------|-------------|
| **L1** | 「关联对象」| 业务能为对象添加关联字段，跨对象引用 |
| **L2** | 「高级列表」| 业务能用搜索/排序/过滤/分页/列控制/导出 |
| **L3** | 「导入导出」| 业务能导入/导出实例数据 (CSV) 和应用模板 (JSON) |
| **L4** | 「权限与布局」| 业务能用行级权限隔离数据 + 多列栅格表单 |

每个 Epic 完成后 = 一次 demo 节点 + 一次发版内发布（alpha → beta → rc）。

## 二、Epic L1：关联对象（用户故事与 AC）

### US-201：添加关联字段

**作为** 业务用户
**我希望** 在"订单"对象里添加一个"客户"字段，关联"客户"对象
**以便于** 每个订单都关联到一个真实客户，无需重复录入客户信息

#### 验收标准（Given/When/Then）

```gherkin
Given 我打开了"订单管理"应用 → "订单"对象 → 字段列表
When 我点击"添加字段" → 选择类型"关联" → 选择目标对象"客户" → 选择显示字段"客户名称"
And 我点击"保存"
Then 新字段"客户"显示在字段列表中，类型为"lookup"
And 字段配置中显示 "关联对象=客户 (customer), 显示字段=name"

When 我创建一个新订单 → 填写订单信息
Then "客户"字段是一个下拉框，列出所有客户的名称
And 我选择一个客户后，订单详情显示客户的 ID 和名称
```

#### 详细 AC

| # | 条件 |
|---|------|
| AC-201.1 | AppObject Service 支持新字段类型 `lookup`，字段配置含 `lookup.objectId` 和 `lookup.displayField` |
| AC-201.2 | 添加 lookup 字段时，后端校验目标对象必须存在；删除 lookup 字段不影响已存在数据 |
| AC-201.3 | lookup 字段在数据库中存为 BIGINT（外键 ID），DDL 自动生成 |
| AC-201.4 | 表单渲染 lookup 字段时，渲染为下拉框，下拉选项来自 `GET /api/apps/{appId}/objects/{lookupObjectId}/instances?page=1&size=50` |
| AC-201.5 | 列表页显示 lookup 字段时，显示关联对象的 displayField 值（连表查询） |
| AC-201.6 | 关联对象被删除时，lookup 字段值显示 "(已删除)" 不报错 |
| AC-201.7 | 一个对象最多关联 5 个 lookup 字段（防止过深嵌套） |

### US-202：通过关联字段过滤

**作为** 业务用户
**我希望** 在"客户"对象列表页，能筛选该客户关联的所有订单
**以便于** 我能快速找到某个客户的历史订单

#### 验收标准

```gherkin
Given 我打开了"客户"对象 → 客户 ID=123 的详情页
When 我点击"查看该客户的订单"按钮
Then 我跳转到"订单"对象列表页，自动应用过滤器 "客户 = 123"
And 列表显示该客户的所有订单
```

#### 详细 AC

| # | 条件 |
|---|------|
| AC-202.1 | 对象详情页的 lookup 反向链接自动生成 "查看关联的 [对象名]" 按钮 |
| AC-202.2 | URL 参数支持 `?filter=customer_id=123` 直接应用过滤 |
| AC-202.3 | 列表页头部显示当前过滤器 chip，点击 chip 可移除 |

## 三、Epic L2：高级列表（用户故事与 AC）

### US-203：列表页搜索 + 排序 + 过滤

**作为** 业务用户
**我希望** 在订单列表页能快速找到特定订单
**以便于** 我无需滚动整个列表

#### 验收标准

```gherkin
Given 我打开了"订单"对象列表页
When 我在搜索框输入"北京"
Then 列表实时过滤，只显示订单号或客户名称包含"北京"的订单
And 头部显示"共 23 条 / 过滤自 100 条"

When 我点击"金额"列头
Then 列表按金额升序排序，再次点击降序
And 列头显示排序箭头 ↑/↓

When 我点击"过滤器"按钮 → 添加过滤器 "状态 = 已完成 AND 创建时间 > 2026-01-01"
Then 列表应用复合过滤器
And 过滤器可保存为"我的视图"
```

#### 详细 AC

| # | 条件 |
|---|------|
| AC-203.1 | 顶部搜索框对所有 string 类型字段做不区分大小写的包含匹配 |
| AC-203.2 | 每个列头点击切换排序：未排序 → 升序 → 降序 → 未排序 |
| AC-203.3 | 过滤器支持 =, !=, >, <, >=, <=, contains, in 7 种操作符 |
| AC-203.4 | 过滤器条件可保存到 localStorage 为"我的视图"（最多 5 个） |
| AC-203.5 | 排序 + 过滤 + 搜索 + 分页 4 个参数可同时生效 |
| AC-203.6 | URL 同步所有过滤器状态，刷新页面或分享链接保持一致 |

### US-204：列表页列控制 + 导出

**作为** 业务用户
**我希望** 自定义显示哪些列，并能导出当前数据
**以便于** 我能把数据导入 Excel 做进一步分析

#### 验收标准

```gherkin
Given 我打开了"订单"对象列表页
When 我点击右上角"列设置"图标
Then 弹出列控制面板，列出所有字段（已勾选=可见，未勾选=隐藏）

When 我取消勾选"备注"字段，关闭面板
Then "备注"列从列表中隐藏

When 我拖拽"客户"列到第一列位置
Then 列顺序调整，新位置持久化到 localStorage

When 我点击"导出"按钮 → 选择"导出当前页 (CSV)"
Then 浏览器下载 "订单_2026-07-13_143022.csv"
And 文件包含表头（字段名）+ 当前页所有行
```

#### 详细 AC

| # | 条件 |
|---|------|
| AC-204.1 | 列显示/隐藏可立即生效，无需刷新 |
| AC-204.2 | 列顺序拖拽支持，HTML5 drag-and-drop API |
| AC-204.3 | 列设置保存到 localStorage（key=`list-cols:{objectId}`），按用户隔离 |
| AC-204.4 | CSV 导出使用 UTF-8 BOM 编码，Excel 打开不乱码 |
| AC-204.5 | CSV 导出支持 3 种范围：当前页 / 当前过滤器全部 / 全部（>10000 时提示） |
| AC-204.6 | lookup 字段在 CSV 中显示 displayField 值，不是 ID |
| AC-204.7 | 导出文件名包含应用名 + 对象名 + 时间戳 |

## 四、Epic L3：导入导出（用户故事与 AC）

### US-205：CSV 文件导入

**作为** 业务用户
**我希望** 下载一个 CSV 模板 → 在 Excel 里填好数据 → 上传导入
**以便于** 我能批量录入历史订单

#### 验收标准

```gherkin
Given 我打开了"订单"对象 → 导入页
When 我点击"下载 CSV 模板"
Then 浏览器下载 "订单模板.csv"，包含表头行 + 1 行示例数据

When 我点击"选择文件" → 上传我的 "订单_2026Q3.csv" (含 500 行)
Then 系统解析 CSV，弹出"字段映射"对话框:
  - 第一列 "订单号" → 映射到字段 order_no
  - 第二列 "客户名称" → 映射到字段 customer_id (显示 lookup)
  ...

When 我确认映射 → 点击"开始导入"
Then 显示进度条 (1/500 ... 500/500)
And 导入完成后显示摘要: "成功 498 条, 失败 2 条, 失败原因: ..."
And 失败行可下载"错误报告.csv"
```

#### 详细 AC

| # | 条件 |
|---|------|
| AC-205.1 | CSV 模板下载按钮每次下载含全部字段（含 lookup 字段） |
| AC-205.2 | 字段映射对话框对未匹配的列给出红色警告 |
| AC-205.3 | lookup 字段导入时支持 "按 ID" 或 "按 displayField" 两种匹配模式 |
| AC-205.4 | 必填字段为空 → 该行失败但不影响其它行 |
| AC-205.5 | 导入使用 dry-run 模式先校验所有行，再提交事务；任一行失败可选择"跳过错误行"继续 |
| AC-205.6 | 导入进度每 100 行更新一次 UI（节流） |
| AC-205.7 | 单次导入上限 10000 行（超过提示拆批） |
| AC-205.8 | 错误报告 CSV 包含：原行号、原始数据、错误原因 |

### US-206：应用模板导出/导入

**作为** 业务实施顾问
**我希望** 把"客户关系管理"应用打包成 JSON 文件，在另一个环境导入
**以便于** 我能快速复制应用结构到客户的生产环境

#### 验收标准

```gherkin
Given 我打开了"客户关系管理"应用 → 设置
When 我点击"导出应用模板"
Then 浏览器下载 "crm-template_v1.0.2_2026-07-13.json"
And JSON 文件包含: 应用基本信息 + 全部对象 schema + 全部页面 schema + 全部流程定义

When 我在新环境 → "导入应用模板" → 上传该 JSON
Then 系统显示"导入预览": 将创建 1 个应用、3 个对象、8 个页面、2 个流程
And 冲突检查: code=crm 已存在 → 提示"重命名或覆盖"

When 我选择"重命名为 crm_v2" → 点击"确认导入"
Then 新应用创建完成，所有对象、页面、流程就绪
And 表单和列表立即可用
```

#### 详细 AC

| # | 条件 |
|---|------|
| AC-206.1 | 应用模板 JSON 含版本号 `version: "v1.0.2"`，便于向后兼容 |
| AC-206.2 | 模板可选择性"含数据 / 仅结构"（默认仅结构） |
| AC-206.3 | 导入时校验版本兼容性：v1.0.1 模板可导入 v1.0.2（升级），反之提示降级风险 |
| AC-206.4 | 导入冲突时 4 种策略：拒绝、重命名、覆盖、合并 |
| AC-206.5 | 模板文件大小 > 5MB 时给出友好提示（不让浏览器卡死） |

## 五、Epic L4：权限与布局（用户故事与 AC）

### US-207：行级权限

**作为** 业务管理员
**我希望** 配置"销售员只能看到自己创建的订单"
**以便于** 数据按角色自动隔离，无需业务侧手工过滤

#### 验收标准

```gherkin
Given 我打开了"订单管理"应用 → 设置 → 行级权限
When 我配置: 角色=销售员, 数据范围=本部门 (dept_id = 当前用户 dept_id)
Then 销售员登录后，列表页只显示本部门的订单
And 销售员尝试通过 URL 访问 dept_id=其他部门 的订单 → 403 禁止

When 我配置: 角色=总经理, 数据范围=全部
Then 总经理登录后，能看到所有订单
```

#### 详细 AC

| # | 条件 |
|---|------|
| AC-207.1 | 行级权限策略 4 种：本用户 (owner_id=me)、本部门、自定义表达式、全部 |
| AC-207.2 | 自定义表达式支持类似 SQL 的 DSL：`owner_id = ${currentUser.id} OR dept_id = ${currentUser.deptId}` |
| AC-207.3 | 行级权限在 Service 层强制应用，绕过 UI 也无法访问（GET / 列表 / 详情 / 编辑 / 删除 5 个端点） |
| AC-207.4 | 权限策略应用后返回行数 < 100 时正常返回；=0 时返回空列表（不报错） |
| AC-207.5 | 行级权限与 v1.0.1 字段级权限（AC-302.6）叠加生效 |

### US-208：多列布局表单

**作为** 业务用户
**我希望** 在画表单时把字段分成 2 列/3 列布局
**以便于** 长表单不滚动过多，PC 端屏幕利用率高

#### 验收标准

```gherkin
Given 我打开了表单编辑器 → "出差报销"表单
When 我点击工具栏"布局" → 选择"2 列"
Then 编辑器切换到 2 列栅格，拖入字段自动填满当前列

When 我把"出差事由"字段从第 1 列拖到第 2 列
Then 字段重新布局，第 1 列的下方字段自动上移填空

When 我预览表单
And 在 1920x1080 屏幕查看，表单显示为 2 列
And 在 768px 宽度查看，表单自动变为 1 列（响应式）
```

#### 详细 AC

| # | 条件 |
|---|------|
| AC-208.1 | 表单布局支持 1 / 2 / 3 / 4 列 4 种模式（修复 AC-201.5 限制） |
| AC-208.2 | 每个字段可指定跨列数（span=1, 2, 3, 4） |
| AC-208.3 | 布局切换不丢失已添加字段，自动按当前顺序排列 |
| AC-208.4 | 移动端 (<768px) 自动降级为 1 列布局 |
| AC-208.5 | 字段顺序拖拽在多列模式下同样可用 |
| AC-208.6 | 已有的 v1.0.1 表单默认 1 列布局，向后兼容 |
