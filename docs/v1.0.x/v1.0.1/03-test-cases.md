# v1.0.1 端到端测试用例

> **版本**：v1.0.1「Forge+Flow」
> **配套文档**：[`01-user-stories.md`](./01-user-stories.md) · [`04-test-report-template.md`](./04-test-report-template.md)
> **目的**：列出 v1.0.1 全部必须通过的测试用例，作为自动化 + 手工测试的依据。

---

## 〇、用例编号规范

```
TC-{Epic}-{编号}

例：TC-M1-001 = M1 Epic 的第 1 个用例
    TC-NF-001 = 非功能（性能/安全）的第 1 个用例
    TC-INT-001 = 集成的第 1 个用例
```

**类型**：
- **E2E**（End-to-End）：跨前后端，使用 Playwright/Cypress
- **Integration**：使用 Jest + supertest 跨服务调用
- **Unit**：单文件级别（Jest/RTL）
- **Security**：渗透 + 越权 + 注入
- **Performance**：压测（k6 或 JMeter）

## 七、自动化测试运行

### 7.1 单测

```bash
# 后端
cd metaplatform-app-service
npm test -- --coverage

# 前端
cd metaplatform-frontend
npm test -- --coverage
```

### 7.2 E2E（Playwright）

```bash
# 启动全栈
cd metaplatform-app-service && npm run dev &
cd metaplatform-frontend && npm run dev &
cd metaplatform-ontology-engine && mvn spring-boot:run &

# 跑 E2E
npm run e2e

# 报告
open test-reports/playwright-report/index.html
```

### 7.3 性能压测

```bash
cd tests/performance
k6 run submit-form.js --vus 100 --duration 30s
```
