# MetaPlatform 菜单产品设计 v2

> **文档版本**：v2（彻底重构）
> **核心原则**：**一级菜单 = 独立功能中心（一项一职，不重叠）**
> **设计理念**：**直接、扁平、可视化**
> **更新日期**：2026-07-03

---

## 〇、v2 设计哲学

### 0.1 v1 的问题反思

| v1 的设计 | 问题 |
|---------|------|
| "我的工作" 一级菜单 | 太抽象，用户不知道点进去能看到什么 |
| 顶部 Tab 二级菜单 | 层级变深，用户认知路径变长 |
| "构建应用" 整合多种能力 | 把建模/流程/页面/智能体混在一起，反而找不到 |
| "数据智能" 包含问数/指标/数据源 | 驾驶舱 + 后台混淆 |
| 4 类角色分发菜单 | 过于复杂，实际产品不需要这么细 |

### 0.2 v2 的核心原则

1. **每个一级菜单 = 一个独立功能中心**，不重叠、不交叉
2. **菜单命名直白**，用户一眼能懂"点进去是什么"
3. **不在一级菜单里塞"工作台/我的"**，那是 Dashbaord 的事
4. **架构/数据/质量都有可视化**，优先图形化展示
5. **管理类全部进后台管理**，不进业务流

### 0.3 一级菜单 12 项（v2）

| # | 一级菜单 | 聚焦 | 适合角色 |
|---|---------|------|---------|
| 1 | **Dashbaord** | 用户数据 + 用户信息 + 任务待办 + 决策事项 | 全部 |
| 2 | **SuperAI** | AI 对话入口（自然语言处理事务/问答） | 全部 |
| 3 | **架构中心** | 企业 EA（4A）架构：业务/应用/数据/技术架构 | 架构师/领导 |
| 4 | **应用中心** | 构建应用：业务数据建模 + 构建页面 + 配置 | 业务/开发 |
| 5 | **流程中心** | 构建流程：业务流程 + 审批流程 | 业务/开发 |
| 6 | **数据中心** | 数据指标看板 + 指标定义 + 数据湖仓 | 业务/开发/领导 |
| 7 | **本体引擎** | 本体 8 要素：对象/属性/关系/动作/函数/规则/安全/治理发布 | 架构师/开发 |
| 8 | **质量中心** | AI Bug 测试 + 自动化测试 + 流程测试 | 测试/开发 |
| 9 | **知识库** | 文档管理 + RAG + GraphRAG | 全部 |
| 10 | **云市场** | 本体模板 + Skill + Agent + 工作流 + 知识包 + API | 全部 |
| 11 | **数字员工** | 多智能体协作 + 决策会议 + 数字员工管理 | 全部 |
| 12 | **后台管理** | 用户 + 权限 + 监控 + 日志 + 部署 + 计费 | 运维 |

> **关键设计**：每个一级菜单都是一个**独立功能中心**，点进去直接是该领域的内容，不再做二级菜单聚合。

### 1️⃣ Dashbaord（工作台）

**聚焦**：用户的数据 + 信息 + 任务 + 决策

**核心价值**：登录后的第一屏，**千人千面**，4 类角色看到完全不同的工作台。

```

│ Dashbaord                  角色: 业务用户 [切换]                      │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ┌─ 待办（红色徽章 12）─────────────────┐ ┌─ 数据看板 ──────────┐  │
│  │ ☐ 审批：差旅报销单 #20240702-001    │ │ 销售额（本月）       │  │
│  │ ☐ 处理：客户投诉工单 #C-2024-783   │ │ ¥1,234,567 ▲ 12%   │  │
│  │ ☐ 阅读：季度运营报告              │ │ ────────────────     │  │
│  │ ☐ 决策：流程优化方案 v2.1         │ │ 我的应用使用次数     │  │
│  └────────────────────────────────────┘ │ 47 次（本周）▲ 8%    │  │
│                                          └────────────────────────┘  │
│                                                                      │
│  ┌─ 我的应用（最近使用）────────────────────────────────────────┐  │
│  │ 📋 客户管理  📝 报销审批  📊 销售看板  📦 库存查询  + 更多     │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                      │
│  ┌─ 我的数字员工 ────────────────────────────────────────────────┐  │
│  │ 🤖 小秘（数据助手）  🤖 小审（流程助手）  🤖 小记（会议助手）  │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                      │
│  ┌─ 通知 ──────────────┐  ┌─ 快速入口 ───────────────────────┐    │
│  │ 3 条未读           │  │ + 新建应用  + AI 建模  + AI 问数  │    │

```

**4 类角色看到的内容差异**：

| 角色 | Dashbaord 内容 |
|------|---------------|
| 👑 领导 | 战略指标看板 + 团队数字员工概览 + 待审批 + 关键决策 |
| 💼 业务 | 我的待办 + 我的应用 + 我的数字员工 + 数据卡片 |
| 🔧 开发 | 开发中的应用 + 开发中的智能体 + FDE 入口 + 告警 |
| ⚙️ 运维 | 系统健康总览 + 待处理告警 + 资源使用率 + 快速运维 |

**对应后端**：
- `/api/v1/dashboard/role-based?role=...`
- `/api/v1/dashboard/tasks?userId=...`
- `/api/v1/dashboard/metrics?role=...`

### 3️⃣ 架构中心（EA 4A 架构）

**聚焦**：查看企业 EA 架构（4A）：业务架构 + 应用架构 + 数据架构 + 技术架构

**核心价值**：**架构师主战场**，纯架构信息管理 + 可视化展示。

```
架构中心
│
├── Tab 1: 业务架构（业务流程架构 L1-L6）
│   ├── L1 价值链
│   ├── L2 业务能力
│   ├── L3 业务流程图（核心） ← 与"流程中心"联动
│   ├── L4 业务角色
│   ├── L5 业务事件
│   └── L6 业务对象（业务层面）
│
├── Tab 2: 应用架构
│   ├── 应用全景图（系统/模块/服务）
│   ├── 应用依赖关系
│   ├── 应用-流程映射
│   └── 应用-数据映射
│
├── Tab 3: 数据架构
│   ├── 数据主题域
│   ├── 概念数据模型
│   ├── 逻辑数据模型
│   └── 数据分布（湖仓）
│
├── Tab 4: 技术架构
│   ├── 技术栈总览
│   ├── 部署拓扑图
│   ├── 服务依赖图
│   └── 技术选型矩阵
│
└── Tab 5: 4A 联动视图（核心可视化）
    ├── 业务-应用联动（哪个流程用了哪些应用）
    ├── 应用-数据联动（哪些应用读写哪些数据）
    └── 数据-技术联动（哪些数据存在哪些存储）
```

**4A 架构对应百特搭架构图的 L3 本体层**：
- 业务架构 = 业务流程架构（L1-L6）
- 应用架构 = 应用/系统视图
- 数据架构 = 数据主题域/数据模型
- 技术架构 = 技术栈/部署拓扑

**关键设计**：
- **架构中心是"只读 + 规划"**，**不参与具体实施**
- 实施在「流程中心」「应用中心」「本体引擎」「数据中心」
- 架构中心展示的是**蓝图**，与具体实施联动（如点击流程图节点跳到流程中心）

**对应后端**：
- `/api/v1/ea/business-architecture`
- `/api/v1/ea/application-architecture`
- `/api/v1/ea/data-architecture`
- `/api/v1/ea/technology-architecture`

### 7️⃣ 本体引擎（Ontology）

**聚焦**：本体 8 要素 —— 对象/属性/关系/动作/函数/规则/安全/治理发布

**核心价值**：MetaPlatform 的**核心引擎**，所有业务建模的"元元模型"。

```
本体引擎（Ontology）
│
├── 1️⃣ 对象（Objects）
│   ├── 对象建模管理
│   ├── 增强描述
│   ├── 外部系统对象映射
│   └── AI 对象生成
│
├── 2️⃣ 属性（Properties）
│   ├── 字段属性（25 字段类型）
│   ├── 字段语义
│   ├── 附件 RAG 知识
│   └── AI 字段生成
│
├── 3️⃣ 关系（Links）
│   ├── 对象表关系（1:1/1:N/N:N）
│   ├── 对象流关系
│   ├── 语义化关系
│   └── AI 关系推断
│
├── 4️⃣ 动作（Actions）
│   ├── 内部动作
│   ├── 外部动作（API 调用）
│   ├── 业务流程动作
│   └── AI 动作集成
│
├── 5️⃣ 函数（Functions）
│   ├── 业务函数
│   ├── 业务规则函数
│   ├── 服务编排
│   └── AI 规则函数
│
├── 6️⃣ 流程规则（Rules）
│   ├── 业务规则（条件触发）
│   ├── 服务编排
│   └── 决策表
│
├── 7️⃣ 安全（Security）
│   ├── 数据权限（行级/字段级）
│   ├── 动作权限
│   ├── 字段权限
│   └── 敏感数据识别
│
└── 8️⃣ 治理与发布（G&R）
    ├── 本体地图（全景视图）
    ├── 导入导出
```

**关键定位**：
- **本体引擎是 MetaPlatform 的灵魂**，所有"应用中心"的对象建模都基于本体
- **本体引擎 + 应用中心**：本体引擎管"模型"，应用中心管"应用"
- **可视化**：对象关系图、关系图谱、本体地图

**对应后端**：
- `/api/v1/ontology/objects`（对象）
- `/api/v1/ontology/properties`（属性）
- `/api/v1/ontology/relations`（关系）

- `/api/v1/ontology/functions`（函数）
- `/api/v1/ontology/rules`（规则）
- `/api/v1/ontology/security`（安全）
- `/api/v1/ontology/governance`（治理）

## 六、设计原则

### 6.1 v2 的 5 条菜单设计原则

| 原则 | 说明 |
|------|------|
| **1. 一级菜单 = 独立功能中心** | 每个一级菜单是一个独立领域，不重叠、不交叉 |
| **2. 命名直白** | 不抽象、不文艺，用户一眼懂 |
| **3. Dashbaord 永远是第 1 项** | 工作台是一级菜单，是用户登录后的第一屏 |
| **4. 管理类独立** | 后台管理独立成一级菜单，不与业务流混在一起 |
| **5. 数量 ≤ 12 项** | 12 项是认知舒适区的上限 |

### 6.2 v2 与 v1.2 菜单对比

| 维度 | v1.2 | **v2** |
|------|------|------|
| 一级菜单数 | 12 个（但层级混乱） | **12 个（一项一职）** |
| 命名 | 架构中心/数据中心等抽象 | **Dashbaord/SuperAI/架构中心/应用中心/流程中心/数据中心/本体引擎/质量中心/知识库/云市场/数字员工/后台管理** |
| 视觉 | 多级嵌套 | **每个菜单独立页面，内部 Tab 切换** |
| Dashbaord | 没有 | **独立一级菜单（第 1 项）** |
| 架构中心 | 含对象建模等 | **聚焦 4A 架构（业务/应用/数据/技术），可视化** |
| 本体引擎 | 隐式 | **独立一级菜单（第 7 项），8 要素独立展示** |
| 数字员工 | 混在"应用中心 > 智能体" | **独立一级菜单（第 11 项）** |
| 后台管理 | 叫"设置" | **改名"后台管理"（第 12 项）** |

## 七、验收标准

- [ ] 12 个一级菜单，每项都是一个独立功能中心
- [ ] Dashbaord 是登录后第一屏，4 类角色看到不同内容
- [ ] SuperAI 是独立 AI 对话入口
- [ ] 架构中心只展示 4A 架构（业务/应用/数据/技术），不参与实施
- [ ] 应用中心聚焦"构建应用"（业务数据建模 + 构建页面 + 配置）
- [ ] 流程中心聚焦"构建流程"（业务流程 + 审批流程）
- [ ] 数据中心聚焦"指标 + 湖仓"，质量在质量中心
- [ ] 本体引擎独立，展示 8 要素
- [ ] 质量中心独立，含 AI Bug 测试
- [ ] 知识库独立，含 RAG
- [ ] 云市场独立，含 6 类模板
- [ ] 数字员工独立，含多智能体协作
- [ ] 后台管理独立，含用户/权限/监控
- [ ] 每个一级菜单内部用 Tab 切换展示内容，不做二级嵌套
