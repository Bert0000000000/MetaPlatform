## 一、产品定位

### 1.1 一句话定位

> **MetaPlatform = 本体论驱动的 AI 原生应用平台 + 数字员工引擎 + 信创全栈适配**
>
> **致力于企业 AI 智能化转型** —— 让每一家企业拥有自己的数字员工团队

### 1.2 三层核心价值

| 层级 | 价值 | 对标 |
|------|------|------|
| **基础平台** | AI 原生应用平台 | 对标百特搭（70% → 100%） |
| **差异化** | 数字员工 + 本体引擎 + AI 基质 | 领先百特搭 |
| **行业特性** | 信创全栈 + 多租户 + 7 种云 | 满足国央企需求 |

### 1.3 三类用户

| 用户 | 价值主张 | 典型场景 |
|------|---------|----------|
| **领导**（高管/部门负责人） | 看数据、做决策、用数字员工 | AI 看板、AI 问数、战略指标 |
| **业务**（业务人员/PM） | 用应用、跑流程、建模板 | 业务应用、流程审批、模板安装 |
| **开发**（建模师/全栈/低代码） | 建模、开发、构建、运维 | 业务建模、流程设计、应用发布 |

### 4.2 升级要点（vs 旧 9 层架构）

| 升级点 | 详情 |
|--------|------|
| **L1 用户层强化** | 12 个一级菜单 + 72 个 Tab + 5 类角色侧边栏 + ⌘K 全局命令面板 |
| **L2 应用层对应菜单** | L2 严格对应功能清单 F1-F12 的 12 个一级菜单 |

| **L9 信创专门化** | 信创作为 L9 的重要组成部分 |

### 4.3 AI Substrate 作为基质（核心设计哲学）

> **AI 不是单独一层，而是渗透在 L1-L8 每一层中的"基质"**

```
┌──────────────────────────────────────────────┐
│ L1 用户层：AI 对话入口 + ⌘K 命令面板          │

│ L2 应用层：F1-F12 每个模块都有 AI 助手浮窗     │
├──────────────────────────────────────────────┤
│ L3 引擎层：BPMN 有 AI 节点 / 数据栈有 NL2SQL  │
├──────────────────────────────────────────────┤
│ L4 AI 基质层 ⭐（独立但渗透）                 │
│   ├── LLM Gateway                             │
│   ├── Agent Runtime                           │

│ L6 服务层：智能 HTTP / SQL 适配器              │
├──────────────────────────────────────────────┤

├──────────────────────────────────────────────┤
```

| **Elasticsearch 8** | 全文搜索 | F9 知识库、F4 应用中心 |

### 6.2 数据流向

```
业务输入
    ↓
[应用/页面] (L1-L2)
    ↓
[业务对象] (L3) → PostgreSQL
    ↓
[本体引擎] (L3) → Neo4j
    ↓
[数据栈] (L3)
    ├── Doris（OLAP）
    ├── Hudi（湖）
    └── Kafka（事件流）
    ↓
[质量/血缘/AI]
    ↓
[指标/看板] (F6)
```

## 十一、关键技术决策

### 11.1 5 个关键决策

| 决策 | 选项 | 推荐 |
|------|------|------|
| **前端框架** | React 18 / Vue 3 | **React 18 + shadcn/ui + Tailwind**（已落地） |
| **后端语言** | Java / Go / Python | **Java 21 + Spring Boot 3**（主） + **Go**（LLM Gateway） + **Python**（Agent Runtime） |
| **流程引擎** | Flowable / Camunda / Activiti | **Flowable 7**（已落地） |
| **图数据库** | Neo4j / TypeDB / NebulaGraph | **Neo4j 5**（已落地） |
| **AI 模型** | OpenAI / Claude / DeepSeek / Qwen | **多模型路由**（按场景选最优） |

### 11.2 关键技术风险

| 风险 | 影响 | 应对 |
|------|------|------|
| LLM Token 成本失控 | 商业化失败 | LLM Gateway 智能分级 + 预算熔断 |
| 大模型响应延迟 | 用户体验差 | 流式响应 + 客户端缓存 + 模型分级 |
| 多租户数据隔离 | 安全问题 | PG Row-Level Security + 物理隔离 |
| 信创适配工作量 | 8 周延期 | 先兼容达梦/麒麟/鲲鹏，逐步扩展 |
| BPMN 2.0 完整实现 | 20 周工作量 | 优先核心 8 节点 + 3 网关 |
