## 1. 模块概述

### 1.1 模块定位

应用中心是 Mate Platform 的应用管理中心，融合低代码设计器与流程设计器。用户在此完成从"创建应用"到"设计模块"到"发布上线"的全流程，无需编写代码即可搭建业务应用。

### 1.2 目标用户

| 角色 | 说明 | 典型场景 |
|---|---|---|
| 应用管理员 | 负责创建和管理应用 | 创建采购管理应用、配置模块、发布上线 |
| 表单设计者 | 负责设计业务表单 | 拖拽组件搭建采购申请表单 |
| 流程设计者 | 负责设计审批流程 | 配置审批节点、条件分支、会签规则 |
| 业务用户 | 使用已发布的应用 | 填写表单、提交审批、查看数据 |

### 1.3 核心价值

- 零代码搭建：拖拽式表单+流程设计，业务人员可直接上手
- 一站式动线：创建应用→创建模块→设计表单→设计流程→发布，一条线走通
- AI 辅助：SuperAI 辅助生成表单和流程，降低设计门槛
- 应用市场：模板复用，快速创建标准化应用

## 3. 功能详情

### 4.1 应用管理

#### 4.1.1 应用列表页

**入口：** 点击左侧导航栏「应用中心」菜单，或直接访问 `/apphub`

**页面布局：**

- **顶部 Tab 导航：** 我的应用 | 应用市场 | 已发布应用
- **搜索栏：** 位于 Tab 下方左侧，输入框，placeholder="搜索应用名称"，输入后实时过滤列表
- **筛选区：** 搜索栏右侧，下拉筛选「应用分组」（全部/自定义分组）、「应用状态」（全部/设计中/已发布/已下线）
- **操作按钮：** 右上角「创建应用」蓝色主按钮
- **列表区域：** 卡片网格布局，每行 3-4 个卡片（响应式）

**应用卡片内容：**

| 元素 | 说明 |
|---|---|
| 应用图标 | 左上角，48x48 圆角图标 |
| 应用名称 | 图标右侧，加粗 16px |
| 应用编码 | 名称下方，灰色 12px，格式 `APP_XXXX` |
| 应用描述 | 编码下方，灰色 14px，最多显示 2 行 |
| 模块数量 | 卡片底部，格式"3 个模块" |
| 状态标签 | 卡片右上角，标签颜色：设计中(蓝色) / 已发布(绿色) / 已下线(灰色) |
| 更新时间 | 卡片底部右侧，格式"2026-07-16 更新" |

**空状态：** 列表为空时，显示居中引导插画 + 文案"还没有应用，点击创建第一个应用吧" + 「创建应用」按钮

**交互：** 点击应用卡片 → 跳转到应用详情页（4.1.5）

---

#### 4.1.2 创建应用

**入口：** 应用列表页右上角「创建应用」按钮，或空状态页的「创建应用」按钮

**操作步骤：**

1. 用户点击「创建应用」按钮 → 系统弹出创建应用模态对话框
2. 用户在对话框中填写以下表单：

| 字段 | 类型 | 必填 | 校验规则 | 默认值 | 说明 |
|---|---|---|---|---|---|
| 应用名称 | 文本输入 | 是 | 1-50 字符，不能纯空格 | - | 显示在应用卡片和应用详情 |
| 应用编码 | 文本输入 | 是 | 字母+数字+下划线，3-30 字符，全局唯一 | - | 创建后不可修改，作为系统标识 |
| 应用描述 | 多行文本 | 否 | 最多 200 字符 | - | 描述应用用途 |
| 应用图标 | 图标选择器 | 否 | 从预设图标库选择 | 通用应用图标 | 点击弹出图标选择网格 |
| 应用分组 | 下拉选择 | 否 | 选择已有分组或输入新分组名 | 未分组 | 用于列表筛选分类 |

3. 用户点击「保存」按钮 → 系统执行以下校验：
   - 应用名称：非空校验 → 失败则输入框下方红字提示"应用名称不能为空"
   - 应用编码：非空 + 格式 + 唯一性校验 → 失败则提示"应用编码只能包含字母、数字和下划线"或"应用编码已存在"
   - 校验通过 → 调用后端 API 创建应用
4. 创建成功 → 关闭对话框 + 顶部 Toast 绿色提示"应用创建成功" + 应用列表自动刷新，新应用出现在列表第一个
5. 创建失败（网络异常等）→ 顶部 Toast 红色提示"创建失败，请重试" + 对话框保持打开

**按钮：**
- 「取消」灰色按钮：点击关闭对话框，不保存任何内容
- 「保存」蓝色主按钮：点击触发校验和保存

---

#### 4.1.3 编辑应用信息

**入口：** 应用详情页（4.1.5）→ 点击右上角「编辑」按钮

**操作步骤：**

1. 用户在应用详情页点击「编辑」按钮 → 弹出编辑应用模态对话框（与创建应用相同的表单，预填当前应用信息）
2. 应用编码字段显示为灰色只读状态（创建后不可修改）
3. 用户修改需要变更的字段
4. 用户点击「保存」→ 系统校验（同创建应用，跳过编码唯一性校验）
5. 保存成功 → 关闭对话框 + Toast 提示"应用信息已更新" + 详情页信息刷新

---

#### 4.1.4 删除应用

**入口：** 应用详情页 → 点击右上角「更多操作」→ 下拉菜单中选择「删除应用」

**操作步骤：**

1. 用户点击「删除应用」→ 系统弹出确认对话框
2. 对话框内容：红色警告图标 + 文案"确定要删除应用「{应用名称}」吗？删除后所有模块、表单、流程数据将永久删除，无法恢复。"
3. 用户输入应用名称以确认（二次确认，防止误删）
4. 用户点击「确认删除」红色按钮 → 系统删除应用及关联数据
5. 删除成功 → Toast 提示"应用已删除" + 跳转回应用列表页
6. 用户点击「取消」→ 关闭对话框

**约束：**
- 已发布状态的应用不能直接删除，需先下线
- 删除操作不可撤销

#### 4.1.5 应用详情页

**入口：** 应用列表页点击应用卡片，或创建应用成功后自动跳转

**页面路径：** `/apphub/apps/{appId}`

  - 右侧操作按钮组：

    - 「版本管理」灰色按钮 → 版本管理（4.7）
    - 「更多操作」下拉按钮 → 下拉菜单：下线应用、导出应用、删除应用（4.1.4）

- **Tab 导航（信息栏下方）：**
  - 模块列表（默认选中）

  - 搜索框：按模块名称搜索
  - 「创建模块」蓝色按钮：创建新模块（4.2.2）
  - 模块卡片列表：网格布局

**模块卡片内容：**

| 元素 | 说明 |
|---|---|

**交互：** 点击模块卡片 → 进入模块设计器（4.3 或 4.4 或 4.5，取决于模块类型）

### 4.2 模块管理

#### 4.2.1 模块列表

已在应用详情页（4.1.5）中描述。

#### 4.2.2 创建模块

**入口：** 应用详情页 → 模块列表 Tab → 点击「创建模块」按钮

**操作步骤：**

1. 用户点击「创建模块」按钮 → 系统弹出创建模块模态对话框
2. 用户在对话框中填写以下表单：

| 字段 | 类型 | 必填 | 校验规则 | 默认值 | 说明 |
|---|---|---|---|---|---|
| 模块名称 | 文本输入 | 是 | 1-30 字符，不能纯空格 | - | 如"采购申请"、"合同审批" |
| 模块编码 | 文本输入 | 是 | 字母+数字+下划线，2-30 字符，应用内唯一 | - | 创建后不可修改 |
| 模块类型 | 单选按钮组 | 是 | 必选一项 | - | 4 种类型（见下表） |
| 模块描述 | 多行文本 | 否 | 最多 200 字符 | - | 描述模块功能 |
| 模块图标 | 图标选择器 | 否 | 从预设库选择 | 通用图标 | 卡片展示用 |

**模块类型说明：**

| 类型 | 图标 | 设计器 | 用途 |
|---|---|---|---|
| 表单模块 | 表单图标 | 表单设计器（4.3） | 数据采集表单，如采购申请单、报销单 |
| 流程模块 | 流程图标 | 流程设计器（4.4） | 审批流程，如合同审批、请假审批 |
| 看板模块 | 看板图标 | 页面设计器（4.5） | 数据看板，如销售统计、库存监控 |
| 自定义页面 | 页面图标 | 页面设计器（4.5） | 自定义业务页面，如客户列表、项目管理 |

3. 用户点击「保存」→ 系统校验：
   - 模块名称非空 → 失败提示"模块名称不能为空"
   - 模块编码非空 + 格式 + 应用内唯一 → 失败提示"模块编码已存在"或"模块编码只能包含字母、数字和下划线"
   - 模块类型必须选择 → 失败提示"请选择模块类型"
4. 校验通过 → 创建模块 → 关闭对话框 + Toast 提示"模块创建成功" + 模块列表刷新
5. 用户点击「取消」→ 关闭对话框

#### 4.4.4 配置审批节点

3. 点击审批节点 → 右侧属性面板显示配置项：

**基础配置：**

| 配置项 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| 节点名称 | 文本输入 | 是 | "审批" | 如"部门经理审批"、"财务审批" |
| 节点描述 | 多行文本 | 否 | "" | 审批说明 |

**审批人配置：**

| 配置项 | 类型 | 说明 |
|---|---|---|
| 审批人类型 | 单选 | 指定人员 / 指定角色 / 指定部门 / 上级 / 发起人自选 / 表单字段映射 |
| 审批人选择 | 人员选择器 | 当类型为"指定人员"时，从组织架构选择 |
| 角色选择 | 角色选择器 | 当类型为"指定角色"时，选择系统角色 |
| 部门选择 | 部门选择器 | 当类型为"指定部门"时，选择部门 |

**审批方式：**

| 配置项 | 类型 | 说明 |
|---|---|---|

|---|---|---|---|

| 拒绝 | 开关 | 开启 | 显示"拒绝"按钮，拒绝后流程结束 |

#### 4.4.5 配置条件分支

**操作步骤：**

1. 用户从左侧面板拖拽「条件分支」节点到画布中
2. 系统自动生成分支结构：1 个入口 → 2 个分支出口（可增加分支）
3. 点击条件分支节点 → 右侧属性面板：

| 配置项 | 类型 | 说明 |
|---|---|---|
| 节点名称 | 文本输入 | 默认"条件判断" |
| 分支列表 | 可增删 | 默认 2 个分支，点击「添加分支」增加 |

**分支条件配置（每个分支）：**

| 配置项 | 类型 | 说明 |
|---|---|---|
| 分支名称 | 文本输入 | 如"金额>10000"、"金额≤10000" |
| 条件组 | 条件构建器 | 可添加多个条件，支持 AND/OR 逻辑 |

**条件构建器：**

1. 选择表单字段（下拉选择关联表单中的字段）
2. 选择比较运算符（等于/不等于/大于/小于/大于等于/小于等于/包含/为空/不为空）
3. 输入比较值（根据字段类型自动适配输入方式：文本输入/数字输入/日期选择/选项选择）
4. 点击「添加条件」可增加更多条件，条件间支持 AND（并且）/ OR（或者）切换
5. 也可点击「切换为规则引擎」→ 引用 TECH-RULE 中定义的业务规则

**其他分支：** 最后一个分支可设为"其他情况"（默认分支），当所有条件分支都不满足时走此分支

**操作步骤：**

1. 拖拽「抄送节点」到画布

3. 点击节点 → 右侧属性面板：

| 配置项 | 类型 | 说明 |
|---|---|---|
| 节点名称 | 文本输入 | 默认"抄送" |
| 抄送人 | 多选 | 指定人员/指定角色/指定部门/表单字段映射 |
| 抄送内容 | 多行文本 | 抄送通知中附带的信息 |

**说明：** 抄送节点不阻塞流程，发出抄送通知后自动流转到下一个节点

#### 4.4.7 绑定表单与流程

**操作步骤：**

1. 在开始节点配置中（4.4.3）选择「关联表单」→ 下拉显示当前应用中所有表单模块
2. 选择一个表单模块 → 系统加载表单字段列表
3. 表单字段可在以下位置使用：
   - 条件分支的判断条件中（4.4.5）
   - 审批节点的审批人映射中（表单字段映射类型）
4. 流程发布后，用户发起流程时 → 填写关联表单 → 提交后流程自动流转

**联动效果：**
- 用户在终端发起流程 → 填写表单 → 点击"提交"
- 表单数据传入流程引擎
- 流程按配置自动流转到第一个审批节点
- 审批人在待办中看到表单数据和审批按钮

#### 4.4.8 保存流程

**操作步骤：**

1. 用户点击顶部工具栏「保存」按钮
2. 系统校验：
   - 有且仅有一个开始节点和一个结束节点
   - 所有节点之间已连线，无悬空节点
   - 审批节点已配置审批人
   - 条件分支至少有 2 个分支出口
   - 开始节点已关联表单（如应用中有表单模块）
3. 校验通过 → 保存流程配置 → Toast"流程保存成功"
4. 校验失败 → Toast 红色提示具体问题，如"节点「部门经理审批」未配置审批人"

---

#### 4.4.9 测试流程

**操作步骤：**

1. 用户点击顶部工具栏「测试」按钮
2. 系统弹出测试面板（右侧抽屉）
3. 测试面板显示：
   - 模拟表单填写（按关联表单渲染输入项）
   - 用户填写测试数据
4. 点击「发起测试」→ 系统模拟流程流转
5. 测试面板显示流程流转路径：开始 → 节点1(当前) → 节点2 → 结束
6. 当前节点高亮显示，可点击「模拟审批-同意」或「模拟审批-拒绝」
7. 流程走完后显示完整流转路径和每个节点的处理结果
8. 点击「关闭测试」退出测试模式

### 4.6 应用发布

#### 4.6.1 发布应用

**入口：** 应用详情页 → 点击右上角「发布」按钮

**前提条件：** 应用至少有 1 个已配置完成的模块

**操作步骤：**

1. 用户点击「发布」按钮 → 系统弹出发布确认对话框
2. 对话框显示：

| 内容 | 说明 |
|---|---|
| 当前版本 | 如 v1.0（首次发布）或 v1.2（后续发布） |
| 新版本号 | 自动递增，如 v1.1 或 v2.0 |
| 版本类型 | 单选：小版本（Bug修复/小优化）/ 大版本（功能新增/重大变更） |
| 变更说明 | 多行文本，必填，描述本次发布内容 |

3. 系统执行发布前校验：

| 校验项 | 失败提示 |
|---|---|
| 应用至少有 1 个模块 | "请先创建至少一个模块" |
| 表单模块已配置组件 | "模块「{名称}」表单未配置任何组件" |
| 流程模块已配置完整 | "模块「{名称}」流程未配置审批节点" |
| 流程已关联表单 | "模块「{名称}」流程未关联表单" |
| 无未保存的修改 | "有未保存的修改，请先保存" |

4. 校验通过 → 用户点击「确认发布」→ 系统执行发布
5. 发布成功 → Toast"应用发布成功，版本 v1.0" → 应用状态变为「已发布」
6. 发布后效果：
   - 应用出现在 Dashboard 导航栏
   - 已发布模块对业务用户可见可用
   - 流程模块可在待办中心发起

#### 4.6.2 下线应用

**入口：** 应用详情页 → 「更多操作」→「下线应用」

**操作步骤：**

1. 点击「下线应用」→ 弹出确认对话框
2. 文案"下线后用户将无法访问此应用，进行中的流程不受影响。确定下线「{应用名称}」？"
3. 点击「确认下线」→ 应用状态变为「已下线」→ Dashboard 导航移除
4. 点击「取消」→ 关闭对话框

### 4.7 版本管理

#### 4.7.1 版本列表

**入口：** 应用详情页 → 点击「版本管理」按钮

**版本列表：**

#### 4.7.2 回滚版本

**操作步骤：**

2. 弹出确认对话框"回滚到 v1.0 后，当前设计将恢复到该版本状态，未发布的修改将丢失。确定回滚？"

3. 差异内容：模块增删、表单组件变更、流程节点变更
4. 新增内容绿色高亮，删除内容红色高亮，修改内容黄色高亮
5. 点击「关闭」退出对比视图

### 4.9 AI 辅助开发

#### 4.9.1 AI 生成表单

**入口：** 表单设计器顶部工具栏 → 点击「AI辅助」按钮

**操作步骤：**

1. 用户点击「AI辅助」→ 右侧弹出 AI 对话面板
2. 用户在输入框中描述需求，例如："创建一个采购申请表单，包含供应商名称、采购物品明细（物品名称、数量、单价、金额自动计算）、采购总金额、预计交付日期、附件"
3. 用户点击「发送」→ AI 分析需求 → 生成表单方案
4. AI 在对话面板中展示生成的表单预览：
   - 组件列表（每个组件的标签、类型、配置）
   - 预览渲染效果
5. 用户可选择：
   - 点击「应用到画布」→ 将 AI 生成的组件加载到设计器画布中（覆盖当前画布内容，如有内容则提示确认）
   - 点击「重新生成」→ 重新描述需求让 AI 再生成
   - 点击「修改需求」→ 补充说明让 AI 调整，如"增加一个紧急程度字段，选项为高/中/低"
6. 应用后用户可继续手动调整组件

#### 4.9.2 AI 生成流程

**入口：** 流程设计器顶部工具栏 → 点击「AI辅助」按钮

**操作步骤：**

1. 用户点击「AI辅助」→ 右侧弹出 AI 对话面板
2. 用户描述需求，例如："创建一个采购审批流程：发起人提交 → 部门经理审批 → 金额大于10000时总经理审批 → 财务确认 → 结束"
3. AI 生成流程方案：
   - 节点列表和连线关系
   - 每个节点的配置建议
   - 条件分支规则
4. AI 在对话面板展示流程预览图
5. 用户点击「应用到画布」→ 流程节点和连线加载到设计器
6. 用户可手动调整节点配置（审批人、审批方式等）

├── status: enum (draft / configured)
├── createdAt: datetime

AppTemplate

├── coverImage: string
