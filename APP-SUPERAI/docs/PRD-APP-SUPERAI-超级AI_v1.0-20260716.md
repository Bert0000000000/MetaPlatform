## 1. 模块概述

### 1.1 模块定位

APP-SUPERAI 是 Mate Platform 的统一 AI 交互入口，以自然语言对话为核心交互形态，贯穿全平台所有业务场景。用户通过一个对话窗口即可完成知识问答、数据分析、业务操作、关系探索、代码生成和任务编排等全部操作。

APP-SUPERAI 基于 Ant Design X 2.0 的 AI 对话组件构建前端界面，后端融合 RAG 检索、NL2SQL、Action 执行、Ontology 语义推理、Agent 编排等能力，通过 TECH-LLMGW 统一接入大模型，实现"用自然语言驱动企业运营"的产品愿景。

### 1.2 目标用户

| 角色 | 典型场景 | 核心诉求 |
|------|----------|----------|
| 业务用户 | 查询知识库、了解业务规则 | 快速获取准确答案，附带来源引用 |
| 数据分析师 | 自助式数据查询与可视化 | 自然语言驱动查询，自动生成图表 |
| 管理者 | 了解运营状况、发起业务操作 | 对话式下达指令，确认后执行 |
| 开发者 | 低代码应用开发辅助 | AI 生成表单/流程配置，提升开发效率 |
| 运营人员 | 编排多步骤任务、批量操作 | 复杂任务自动分解，执行过程可追踪 |

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 自然语言交互 | 用户无需学习复杂操作界面，用自然语言即可完成所有操作 |
| 知识增强 | 基于 RAG 检索企业知识库，回答准确且可溯源 |
| 数据驱动 | NL2SQL 将自然语言转为 SQL，实现自助式数据分析 |
| 行动闭环 | 从"对话"到"执行"闭环，AI 识别意图后触发 Action 并反馈结果 |
| 语义理解 | 基于 Ontology 本体引擎，理解企业业务语义和数据关系 |
| 任务编排 | 复杂需求自动分解为子任务，并行执行，结果聚合 |
| 数字员工调度 | 识别专业领域内容后自动委托数字员工执行，汇总各方反馈后整合最终答案 |

### 1.4 设计原则

1. **对话优先**：所有功能均通过对话界面入口，不单独开设独立页面
2. **流式反馈**：AI 回答采用流式输出，首字响应 < 2s
3. **可溯源**：RAG 回答必须附带引用来源，Action 执行必须记录审计日志
4. **安全可控**：危险操作需用户二次确认，SQL 需通过安全审计
5. **渐进式复杂度**：普通用户直接提问即可，高级用户可通过 `/` 命令和参数调节控制行为

### 2.3 模式说明

SuperAI 支持 7 种对话模式，用户可通过输入框下方模式切换标签手动指定，也可由 AI 自动识别：

| 模式 | 标识色 | 触发方式 | 典型示例 |
|------|--------|----------|----------|
| 问答 | 蓝色 | 默认模式 / 自动识别 | "公司的差旅报销标准是什么？" |

| 代码 | 绿色 | 手动切换 | "生成一个客户信息登记表单" |
| 任务 | 红色 | 手动切换 / 自动识别 | "分析上个季度的客户流失原因并生成报告" |
| 调度 | 金色 | 手动切换 / 自动识别 | "帮我分析这份合同的法律风险并评估财务影响" |

## 3. 功能详情

### 3.1 智能问答

> 功能编号：FR-AI-001
> 优先级：P0（Phase 1 交付）
> 依赖：TECH-LLMGW、TECH-RAG

#### 3.1.1 功能入口

- **入口 1**：点击顶部导航栏「SuperAI」图标 → 默认进入问答模式
- **入口 2**：在任意页面点击右下角悬浮 AI 按钮 → 弹出问答侧边栏
- **入口 3**：在会话列表点击已有问答会话 → 恢复历史对话

#### 3.1.2 对话界面布局

基于 Ant Design X 2.0 的 `Bubble`、`Sender`、`Conversations` 组件构建。

**界面元素**：

| 区域 | 组件 | 说明 |
|------|------|------|
| 会话列表 | `Conversations` | 左侧面板，展示历史会话，支持搜索、置顶、删除 |
| 消息区域 | `Bubble.List` | 右侧主区域，展示对话消息流，用户消息右对齐，AI 消息左对齐 |
| 输入区域 | `Sender` | 底部输入框，支持多行文本、附件上传、快捷操作 |
| 模式切换 | `Tabs` | 输入框上方标签栏，切换对话模式 |

#### 3.1.3 操作步骤：发起一次智能问答

**前置条件**：用户已登录 Mate Platform，拥有 SuperAI 访问权限。

**操作步骤**：

1. **打开 SuperAI**
   - 用户点击顶部导航栏「SuperAI」图标
   - 系统响应：页面跳转到 SuperAI 主界面，左侧展示会话列表，右侧展示空白对话区域，输入框获得焦点
   - 反馈提示：对话区域展示欢迎语"您好，我是 SuperAI 助手，有什么可以帮您？"

2. **新建会话**（可选，系统默认已新建）
   - 用户点击左侧会话列表顶部的「+ 新建会话」按钮
   - 系统响应：创建新会话，会话列表新增一条"新对话"记录（高亮选中），右侧对话区域清空
   - 反馈提示：输入框获得焦点，光标闪烁等待输入

3. **输入问题**
   - 用户在底部输入框中输入问题文本，例如"公司的差旅报销标准是什么？"
   - 系统响应：输入框实时展示输入内容，支持多行输入
   - 交互详情：输入框右侧出现字符计数（如 18/2000），超过限制时输入框边框变红并提示"最多输入 2000 字符"

4. **发送消息**
   - 用户点击输入框右侧「➤」发送按钮，或按 `Ctrl + Enter` / `Cmd + Enter` 快捷键
   - 系统响应：
     - 用户消息立即显示在对话区域（右对齐气泡），输入框清空
     - AI 消息气泡出现（左对齐），显示"正在思考..."加载动画
     - 输入框下方出现"停止生成"按钮（红色）
   - 反馈提示：AI 气泡内出现打字光标，开始流式输出回答内容

5. **查看流式回答**
   - AI 回答以流式方式逐字输出到 AI 消息气泡中
   - 系统响应：内容实时渲染为 Markdown 格式（支持标题、列表、表格、代码块、加粗等）
   - 反馈提示：回答完成后，"停止生成"按钮消失，AI 气泡底部出现操作栏

6. **查看引用来源**
   - AI 回答完成后，消息气泡底部出现「📚 引用来源」区域
   - 系统响应：展示 1-5 条引用卡片，每条包含：来源标题、文档类型图标、匹配度（如 92%）、内容摘要（前 100 字）
   - 操作步骤：
     1. 用户点击某条引用卡片
     2. 系统响应：右侧弹出文档预览侧边栏，高亮显示引用段落
     3. 用户可点击侧边栏右上角「在新窗口打开」按钮查看完整文档
     4. 用户点击侧边栏右上角「✕」关闭预览
   - 反馈提示：引用卡片点击时有涟漪动画效果

7. **追问（多轮对话）**
   - 用户在输入框中继续输入追问内容，例如"那出差的住宿标准呢？"
   - 系统响应：AI 基于上下文理解追问意图，结合前文继续回答
   - 交互详情：追问消息上方显示"↳ 追问"标记，表示关联上一轮对话
   - 反馈提示：同步骤 5，流式输出回答

8. **消息反馈**
   - AI 回答下方操作栏出现「👍」和「👎」按钮
   - 操作步骤：
     1. 用户点击「👍」按钮 → 系统记录正面反馈，按钮变为已选中状态（蓝色填充）
     2. 用户点击「👎」按钮 → 系统弹出反馈弹窗，包含：问题类型下拉（回答不准确 / 信息过时 / 格式问题 / 其他）、补充说明文本框
     3. 用户填写反馈后点击「提交」按钮 → 弹窗关闭，按钮变为已选中状态（橙色填充）
   - 反馈提示：Toast 提示"感谢您的反馈，将帮助我们持续改进"

9. **复制回答**
   - 用户点击 AI 消息操作栏中的「📋 复制」按钮
   - 系统响应：将回答内容（纯文本格式）复制到剪贴板
   - 反馈提示：Toast 提示"已复制到剪贴板"

10. **保存对话**
    - 系统自动保存：每次消息交互后自动保存到会话历史
    - 手动重命名：用户点击顶部会话标题栏的标题文本 → 变为可编辑输入框 → 输入新标题 → 按 `Enter` 或点击空白处确认
    - 反馈提示：Toast 提示"会话标题已更新"

#### 3.1.4 操作步骤：管理对话历史

1. **查看会话列表**
   - 用户点击左侧面板中的会话列表
   - 系统响应：按时间倒序展示会话列表，每条包含：会话标题、最后消息摘要、时间戳、模式标签
   - 交互详情：会话按时间分组（今天 / 昨天 / 7 天内 / 更早）

2. **搜索历史会话**
   - 用户点击会话列表顶部的「🔍 搜索」输入框
   - 输入关键词后按 `Enter`
   - 系统响应：展示匹配的会话列表（匹配标题和消息内容），关键词高亮
   - 反馈提示：无匹配结果时显示"未找到相关对话"

3. **置顶会话**
   - 用户鼠标悬停在会话列表某条会话上 → 出现操作图标
   - 点击「📌 置顶」图标
   - 系统响应：该会话移动到列表顶部"置顶"区域
   - 反馈提示：无额外提示，会话位置变化即为反馈

4. **删除会话**
   - 用户鼠标悬停在会话列表某条会话上 → 点击「🗑 删除」图标
   - 系统响应：弹出确认弹窗"确定删除此对话？删除后无法恢复"
   - 用户点击「确认」按钮 → 会话从列表中移除
   - 反馈提示：Toast 提示"对话已删除"

5. **导出会话**
   - 用户点击顶部会话标题栏「⋯」更多按钮 → 出现下拉菜单
   - 点击「导出对话」选项
   - 系统响应：弹出导出格式选择弹窗（Markdown / PDF / JSON）
   - 用户选择格式后点击「导出」按钮 → 浏览器下载文件
   - 反馈提示：Toast 提示"正在导出..."→"导出成功"

#### 3.1.5 附件上传（多模态输入）

1. 用户点击输入框左侧「📎」附件按钮
2. 系统响应：弹出文件选择对话框
3. 用户选择文件（支持 .png, .jpg, .pdf, .docx, .xlsx, .txt, .csv，单文件 < 20MB）
4. 系统响应：输入框上方展示文件缩略卡片，包含文件名、文件大小、移除按钮「✕」
5. 用户输入问题文本（如"请总结这个文件的内容"）后点击发送
6. 系统响应：AI 解析文件内容并流式输出回答
7. 反馈提示：文件解析中显示"正在解析文件..."进度提示

#### 3.1.6 快捷命令

用户在输入框中输入 `/` 触发快捷命令面板：

| 命令 | 说明 | 示例 |
|------|------|------|
| `/清空` | 清空当前对话 | - |
| `/模式 [名称]` | 切换对话模式 | `/模式 分析` |
| `/知识库 [名称]` | 指定检索的知识库范围 | `/知识库 财务制度` |
| `/模型 [名称]` | 切换 LLM 模型 | `/模型 doubao-pro` |
| `/温度 [值]` | 调整生成温度 | `/温度 0.3` |

### 3.2 数据分析

> 功能编号：FR-AI-002
> 优先级：P0（Phase 2 交付）
> 依赖：TECH-LLMGW、TECH-ONT（数据语义映射）、StarRocks / PostgreSQL（数据源）

#### 3.2.1 功能入口

- **入口 1**：在 SuperAI 输入框下方模式切换栏点击「分析」标签
- **入口 2**：在问答模式中输入数据分析类问题，AI 自动识别并切换到分析模式
- **入口 3**：在 APP-DASHBOARD 仪表盘页面点击「AI 分析」按钮 → 跳转到 SuperAI 分析模式

#### 3.2.2 操作步骤：自然语言数据查询

**前置条件**：用户已配置数据源访问权限，Ontology 中已定义相关数据表语义映射。

1. **切换到分析模式**
   - 用户点击输入框上方模式切换栏的「分析」标签
   - 系统响应：标签高亮为青色，输入框占位符变为"描述您想分析的数据，如：按部门统计本月销售额"
   - 反馈提示：对话区域顶部出现模式标识条"已切换到数据分析模式"

2. **输入分析需求**
   - 用户在输入框中输入自然语言分析需求，例如"按部门统计本月销售额，按降序排列"
   - 系统响应：输入框正常展示输入内容

3. **发送分析请求**
   - 用户点击「➤」发送按钮
   - 系统响应：
     - 用户消息显示在对话区域
     - AI 气泡出现，显示"正在分析您的需求..."加载状态
     - AI 流式输出分析过程说明："我理解您需要查询各部门本月的销售额并按降序排列。我将从销售订单表中查询..."
   - 反馈提示：分析过程中显示进度条（理解需求 → 生成 SQL → 安全审计 → 执行查询 → 生成图表）

4. **查看生成的 SQL**
   - AI 回答完成后，消息气泡中展示 SQL 预览卡片
   - 系统响应：SQL 以代码块形式展示，支持语法高亮
   - 卡片操作：
     - 「📋 复制」：复制 SQL 到剪贴板
     - 「✏️ 编辑」：进入 SQL 编辑模式
   - 交互详情：SQL 卡片上方显示数据源标识（如"数据源：StarRocks-OLAP"）

5. **编辑 SQL（可选）**
   - 用户点击 SQL 卡片中的「✏️ 编辑」按钮

### 3.7 数字员工调度

> 功能编号：FR-AI-007
> 优先级：P1（Phase 4 交付）
> 依赖：TECH-AGENT、APP-DW、TECH-LLMGW、TECH-RAG、TECH-A2A

#### 3.7.1 功能说明

SuperAI 作为「智能调度中枢」，当用户提出的问题涉及专业领域（如法律、财务、人力资源、采购等）时，自动识别并委托给对应领域的数字员工执行。SuperAI 负责实时展示所有数字员工的任务进展，收集各员工反馈后，整合生成最终回答返回给用户。

**核心机制：**

```
用户提问 → SuperAI 意图识别 → 判断是否需要专业数字员工
  ├─ 通用问题 → SuperAI 直接回答（走问答/分析等模式）
  └─ 专业领域问题 → 进入调度模式
       ↓
  SuperAI 分析问题，拆分为专业子任务
       ↓
  匹配可用数字员工（按能力/知识范围/状态筛选）
       ↓
  向各数字员工委托子任务（通过 TECH-AGENT / TECH-A2A）
       ↓
  实时展示各员工执行进展（页面渲染调度面板）
       ↓
  各员工完成任务并返回结果
       ↓
  SuperAI 汇总所有员工反馈 → LLM 整合生成最终回答
       ↓
  展示最终回答 + 各员工贡献详情 + 追问入口
```

#### 3.7.2 功能入口

- **入口 1**：在模式切换栏点击「调度」标签（金色），手动切换到调度模式
- **入口 2**：在任意模式下输入专业领域问题，AI 自动识别并切换到调度模式
- **入口 3**：在任务编排模式中，AI 识别到某子任务需要专业数字员工时，自动委托

#### 3.7.5 跨系统 Agent 协作（A2A）

当平台内数字员工无法满足需求时，SuperAI 可通过 A2A 协议委托外部 Agent：

1. 在调度计划中，如果某子任务无匹配的内部数字员工
2. AI 提示"未找到内部匹配的数字员工，是否搜索外部 Agent？"
3. 用户点击「搜索外部 Agent」→ 系统通过 TECH-A2A 查询已注册的外部 Agent Card
4. 展示外部 Agent 列表（含 Agent 名称、能力声明、所属组织）
5. 用户选择外部 Agent → 系统通过 A2A 协议委托任务
6. 执行过程与内部数字员工一致（实时展示进度、收集结果）
7. 结果汇总时标注"由外部 Agent [名称] 提供"

## 5. 依赖关系

### 5.1 上游依赖

| TECH-LLMGW | LLM 推理调用、流式输出、模型路由 | 强依赖 | 主模型不可用时自动切换备用模型 |
| TECH-RAG | 知识库检索、文档嵌入、混合检索（向量 + BM25） | 强依赖 | RAG 不可用时退化为纯 LLM 回答（无引用） |
| TECH-ACTION | Action 定义查询、Action 执行 | 强依赖（Action 模式） | Action 模式不可用时提示"操作服务暂不可用" |
| TECH-AGENT | Agent 框架、任务编排、多步推理 | 强依赖（任务模式） | 任务模式不可用时提示"任务编排服务暂不可用" |
| TECH-ONT | Ontology 概念查询、关系推理、数据语义映射 | 强依赖（探索/分析模式） | 探索/分析模式不可用时提示"本体服务暂不可用" |
| TECH-MCP | MCP 工具调用、外部资源访问 | 中等依赖 | MCP 工具不可用时该工具功能降级 |
| TECH-A2A | A2A 协议、外部 Agent 发现与委托 | 中等依赖（调度模式） | A2A 不可用时仅支持内部数字员工 |
| APP-DW | 数字员工列表查询、能力信息、状态获取 | 强依赖（调度模式） | APP-DW 不可用时调度模式不可用，提示"数字员工服务暂不可用" |
| TECH-IAM | 用户权限校验 | 强依赖 | 权限服务不可用时拒绝所有操作 |
| TECH-OBS | 审计日志、链路追踪 | 强依赖 | 可观测不可用时操作正常但记录延迟 |

### 5.2 下游消费方

| 下游方 | 消费内容 | 集成方式 |
|--------|----------|----------|
| APP-APPHUB | AI 辅助开发能力（代码生成、流程模板） | SuperAI 侧边栏嵌入 |
| APP-DW | 数字员工复用 SuperAI 的对话和推理能力；SuperAI 调度数字员工执行专业任务 | API 调用（双向） |
| APP-DASHBOARD | 仪表盘展示 AI 使用统计指标 | 数据接口 |
| APP-ONTSTUDIO | 在本体论引擎中嵌入 SuperAI 探索入口 | 按钮跳转 |

### 5.3 交互流程

```
用户输入消息 → APP-SUPERAI 前端
  ↓
意图识别 → 判断消息类型（问答/分析/操作/探索/代码/任务/调度）
  ↓
路由到对应处理引擎:
  ├─ 问答 → TECH-RAG 检索（Milvus 向量 + BM25 关键词 → 重排序）
  │        → TECH-LLMGW 生成回答（流式）→ 附带引用来源 → 返回前端
  ├─ 分析 → TECH-ONT 数据语义映射 → TECH-LLMGW 生成 SQL
  │        → SQL 安全审计 → 数据源执行查询 → 结果可视化 → 返回前端
  ├─ 操作 → TECH-ACTION 匹配 Action → 参数填充
  │        → 用户确认 → TECH-ACTION 执行 → 结果反馈 → 审计日志
  ├─ 探索 → TECH-ONT 查询 → Neo4j 图查询
  │        → 知识图谱数据 → AntV X6 可视化 → 返回前端
  ├─ 代码 → TECH-LLMGW 生成代码/配置
  │        → 预览渲染 → 用户编辑 → 应用到 APP-APPHUB
  ├─ 任务 → TECH-AGENT 任务分解 → 生成执行计划
  │        → 用户确认 → 多步执行（并行/串行）→ 结果聚合 → 报告生成
  └─ 调度 → 意图识别判断需要专业数字员工
           → AI 拆分为领域子任务 → 匹配数字员工（APP-DW 查询可用员工）
           → 用户确认调度计划 → 通过 TECH-AGENT 向各数字员工委托任务
           → SSE 实时推送执行状态 → 调度面板更新
           → 各员工完成 → 收集结果 → TECH-LLMGW 整合汇总
           → 生成最终回答（标注各员工贡献）→ 返回前端
  ↓
所有执行记录 → PostgreSQL 持久化 → TECH-OBS 审计日志
```

├── naturalLanguageQuery: Text       // 用户原始自然语言查询

├── retryCount: Integer

├── executedAt: Timestamp

OntologyQuery（Ontology查询记录）

├── sessionId: UUID (FK -> ChatSession.id)
├── messageId: UUID (FK -> ChatMessage.id)
├── queryText: Text                  // 自然语言查询
