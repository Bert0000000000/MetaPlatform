# 应用中心页面编辑器 — 架构重构计划

## 一、总结

将 Pages.tsx 和 PageEditor.tsx 中的 5 个内联编辑器组件抽取到 `editors/` 目录下作为独立文件，消除双重实现问题，修复 12 个 Bug，统一编辑体验。

---

## 二、现状问题

| # | 问题 | 严重度 |
|---|------|--------|
| 1 | 两个不同的 FormOrLowCodeEditor 实现（Pages.tsx 内联 vs PageEditor.tsx 内联），UI 和功能不一致 | P0 |
| 2 | 5 个编辑器组件混在 PageEditor.tsx（938行），职责混乱 | P0 |
| 3 | ListPageEditor 右侧配置面板的"保存"按钮无 onClick，配置修改丢失 | P0 |
| 4 | ListPageEditor 有重复的"创建人"列（field: creator 和 field: author） | P0 |
| 5 | 树根节点硬编码"采购需求"，应显示应用名称 | P1 |
| 6 | 移动端预览图标用了 Monitor 而非 Smartphone | P1 |
| 7 | 新建页面对话框仅 3 种类型（LowCode/ProCode/AI），应支持全部 9 种 | P1 |
| 8 | GridLayout 组件已实现但未被任何编辑器使用 | P1 |
| 9 | 拖拽功能不完整（缺少 dataTransfer API） | P2 |
| 10 | FlowEditor 无法拖拽节点、不支持分支连线 | P2 |

## 四、实施步骤

### 阶段 0: 创建 types.ts（共享类型）

**文件**: `editors/types.ts`（新建）

- 导出 `PageComponent`、`PageVersion`、`PageData`、`BaseEditorProps`、`FormEditorProps`
- 导出 `TYPE_META`（10 种页面类型元信息）
- 导出 `PAGE_TYPE_OPTIONS`（9 种可创建类型）
- 导出 `COMPONENT_PALETTE`（10 种组件）

**理由**: 两个文件各自定义了重复的类型，合并后只保留一份。

---

### 阶段 1: 创建 EditorShell（共享工具栏）

**文件**: `editors/EditorShell.tsx`（新建）

从 PageEditor.tsx 第 142-252 行提取共享 UI：
- 顶部工具栏：返回按钮（可选）、页面名输入、类型 Badge、未保存标记
- 版本号按钮 + 版本历史面板
- 设备预览按钮组（**修复 Bug #6**：desktop=Monitor, tablet=Tablet, mobile=Smartphone）
- AI 助手开关
- 保存按钮
- 主内容区 + AI 面板的 flex 布局

Props 控制哪些功能可见（`showBack`、`showAI`），Pages.tsx 内联编辑器和 PageEditor.tsx 全屏编辑器共用此组件。

---

### 阶段 2: 创建 FormLowCodeEditor（合并统一）

**文件**: `editors/FormLowCodeEditor.tsx`（新建）

合并策略：
- 取 Pages.tsx 的 lucide-react 图标组件面板（视觉更好）
- 取 PageEditor.tsx 的 12 列栅格布局逻辑
- **引入 GridLayout + SpanSelector**（修复 Bug #8）
- 使用 `dataTransfer.setData/getData` 实现拖拽（修复 Bug #9 基础版）

Props: `FormEditorProps`（components, setComponents, selectedCompId, setSelectedCompId, setDirty）

---

### 阶段 3: 创建 ListPageEditor（修复 Bug #3 #4）

**文件**: `editors/ListPageEditor.tsx`（新建）

从 PageEditor.tsx 提取，同时修复：
- **Bug #3**: config 状态初始化从 `components[0].props` 读取；"保存"按钮绑定 onClick，变更通过 setComponents 同步
- **Bug #4**: 删除重复的 "创建人" 列（field: "author"）

---

### 阶段 4: 创建 ReportEditor（增强）

**文件**: `editors/ReportEditor.tsx`（新建）

从 PageEditor.tsx 提取，增强：
- 使用 GridLayout 替换内联 grid 逻辑
- widget 卡片添加操作栏（编辑/删除/span 切换）
- 每种 chart 类型显示对应 SVG 占位图
- 状态变更同步回 components

---

### 阶段 5: 创建 FlowEditor（增强）

**文件**: `editors/FlowEditor.tsx`（新建）

从 PageEditor.tsx 提取，增强：
- 节点支持鼠标拖拽（onMouseDown + document mousemove）
- 连接线用贝塞尔曲线替代直线
- 状态变更同步回 components

---

### 阶段 6: 创建 BIEditor（修正命名）

**文件**: `editors/BIEditor.tsx`（新建）

从 PageEditor.tsx 提取，修正：
- 标题改为 "BI & Agent 编辑器"
- 增加"数据看板配置"区域
- 状态变更同步回 components

---

### 阶段 7: 修改 Pages.tsx

1. 导入改为从 `./editors` 导入所有编辑器 + 从 `./editors/types` 导入 TYPE_META 等
2. **修复 Bug #5**: 树根节点从 "采购需求" 改为动态应用名称（从 API 获取）
3. **修复 Bug #6**: 导入 Smartphone 图标，修复移动端预览图标
4. **修复 Bug #7**: 新建页面对话框使用 PAGE_TYPE_OPTIONS（9 种类型）
5. 删除内联 FormOrLowCodeEditor（line 716-786），改为使用导入的 FormLowCodeEditor
6. 删除重复的 TYPE_META 和 COMPONENT_PALETTE 定义
7. 可选：使用 EditorShell 替换内联工具栏

---

### 阶段 8: 修改 PageEditor.tsx

1. 导入改为从 `./editors` 导入所有编辑器
2. 删除第 278-937 行的所有内联编辑器组件定义
3. 使用 EditorShell 替换内联工具栏
4. 从 938 行精简为约 150 行（纯外壳 + 数据加载/保存 + 路由）

---

### 阶段 9: 更新 editors/index.ts

导出所有编辑器组件、类型、栅格系统。
