# 应用中心编辑器模式 — 完善计划

## 一、目标

消除 Pages.tsx 与 PageEditor.tsx 之间的代码重复，统一编辑体验。

---

## 二、当前问题

| # | 问题 | 影响 |
|---|------|------|
| 1 | Pages.tsx (773行) 自己实现了工具栏，未使用 EditorShell | 内联编辑缺少 AI 助手、设备切换等功能 |
| 2 | Pages.tsx 自行定义了 PageComponent/PageVersion 类型 | 与 editors/types.ts 重复 |
| 3 | Pages.tsx 的保存/版本/脏状态逻辑与 PageEditor.tsx 重复 | 维护成本高，行为不一致 |
| 4 | 两个编辑入口（内联 vs 独立页面）行为不完全一致 | 用户体验混乱 |

---

## 三、方案

### 方案 A: Pages.tsx 使用 EditorShell（推荐）

**思路**: 让 Pages.tsx 的右侧编辑区域使用 EditorShell，而非自己重写工具栏。

**改动**:
1. Pages.tsx 从 `editors/types` 导入 PageComponent/PageVersion（删除本地定义）
2. Pages.tsx 的右侧编辑区域用 EditorShell 包裹子编辑器
3. 提取共享的编辑状态管理逻辑为 custom hook `usePageEditor`

**优点**: 改动最小，复用已有组件
**缺点**: Pages.tsx 仍然较重

### 方案 B: 统一为单一编辑入口

**思路**: 删除 Pages.tsx 的内联编辑，点击页面时直接跳转到 PageEditor.tsx。

**改动**:
1. Pages.tsx 右侧只显示页面预览 + "编辑"按钮
2. 点击编辑跳转到 `/apps/:appId/page-editor?pageId=xxx`
3. Pages.tsx 精简为纯树面板

**优点**: 彻底消除重复
**缺点**: 失去内联编辑的便利性（需要跳转页面）

### 方案 C: 提取共享 Hook + EditorShell 复用

**思路**: 提取 `usePageEditor` hook 管理所有编辑状态，Pages.tsx 和 PageEditor.tsx 都使用它。

**改动**:
1. 创建 `editors/usePageEditor.ts` hook
2. Pages.tsx 和 PageEditor.tsx 都使用这个 hook
3. Pages.tsx 使用 EditorShell 渲染工具栏

**优点**: 代码复用最大化，行为完全一致
**缺点**: 需要重构两个文件

---

## 四、推荐方案: 方案 C

**理由**: 彻底解决重复问题，且保留内联编辑的便利性。

---

## 五、实施步骤

### 步骤 1: 创建 usePageEditor hook

**文件**: `editors/usePageEditor.ts`（新建）

提取 Pages.tsx 和 PageEditor.tsx 中重复的逻辑:
- `components` / `setComponents` 状态
- `pageName` / `setPageName` 状态
- `currentVersion` / `versions` 状态
- `dirty` / `saving` 状态
- `device` / `showAI` UI 状态
- `loadPage()` 函数
- `savePage()` 函数
- `restoreVersion()` 函数

```typescript
export function usePageEditor(appId: string, pageId: string | null) {
  // 所有状态和逻辑集中在这里
  return {
    pageData, pageName, setPageName,
    components, setComponents,
    currentVersion, versions,
    dirty, setDirty,
    saving, device, setDevice,
    showAI, setShowAI,
    selectedCompId, setSelectedCompId,
    loadPage, savePage, restoreVersion,
  };
}
```

### 步骤 2: 修改 Pages.tsx

1. 删除本地的 PageComponent/PageVersion 类型定义（第 30-42 行）
2. 从 `./editors/types` 导入这些类型
3. 使用 `usePageEditor` hook 管理编辑状态
4. 右侧编辑区域用 `EditorShell` 包裹子编辑器
5. 删除重复的工具栏实现（第 608-650 行）

### 步骤 3: 修改 PageEditor.tsx

1. 使用 `usePageEditor` hook 管理编辑状态
2. 删除重复的状态定义和逻辑

### 步骤 4: 构建验证

1. `npx vite build` 确保编译通过
2. 测试内联编辑和独立编辑器两种模式
